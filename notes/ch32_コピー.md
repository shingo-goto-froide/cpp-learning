# 第32章: コピー

## 要点

### C++のコピーは値コピー（C#と真逆）

```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b = a;   // 中身が丸ごとコピーされる（3要素コピー）
b[0] = 100;
// a[0] == 1（影響なし、別物になっている）
```

```csharp
// C# の場合
List<int> a = new() {1, 2, 3};
List<int> b = a;          // 参照だけコピー
b[0] = 100;
// a[0] == 100（同じオブジェクトを指すので影響あり）
```

**C++は `=` で中身が丸ごとコピーされる**。これが最大の違い。

### コピーコンストラクタ と コピー代入演算子 は別物

```cpp
class Foo {
public:
    Foo(const Foo& other);             // ①コピーコンストラクタ（初期化時）
    Foo& operator=(const Foo& other);  // ②コピー代入演算子（代入時）
};

Foo a;
Foo b = a;    // ① が呼ばれる（初期化 = コンストラクタ）
Foo c;
c = a;        // ② が呼ばれる（代入 = operator=）
```

ch31 の「初期化と代入の違い」がコピーでも同じく分かれる。**`=` の左に型名があるか**で見分ける。

### 両方とも値コピー

| | コピーコンストラクタ | コピー代入 |
|---|---|---|
| いつ呼ばれる | オブジェクト**作成時** | 既存オブジェクトへの代入時 |
| 元の状態 | まだ構築されていない | すでに何か持っている |
| やること | 他方からコピーして構築 | 既存を破棄→他方からコピー |
| 中身のコピー | ○ | ○ |

**どちらも最終的に「中身の値コピー」**という点は共通。

### デフォルトは浅いコピー（shallow copy）

```cpp
class MyVec {
    int* data;
    size_t size;
    // コピーコンストラクタを書かないと…
};
```

コンパイラが自動生成するコピーはメンバーごとの単純コピー：

```cpp
// b = a; のとき自動生成される動作：
b.data = a.data;   // ポインターをそのままコピー（同じメモリーを指す！）
b.size = a.size;
```

### 浅いコピーの問題

1. `a` と `b` が**同じメモリーを指す**
2. `a` のデストラクタで `delete data` した後、`b` のデストラクタで**二重解放**
3. どちらかが書き換えるともう一方も影響

### 深いコピー（deep copy）の実装

```cpp
class Buffer {
    int* data;
    size_t size;
public:
    // コピーコンストラクタ
    Buffer(const Buffer& o) : size(o.size), data(new int[o.size]) {
        std::copy(o.data, o.data + o.size, data);
    }

    // コピー代入演算子
    Buffer& operator=(const Buffer& o) {
        if (this == &o) return *this;   // 自己代入チェック
        delete[] data;                   // 既存を解放
        size = o.size;
        data = new int[o.size];
        std::copy(o.data, o.data + o.size, data);
        return *this;
    }

    ~Buffer() { delete[] data; }
};
```

### Rule of Three（最重要）

**「デストラクタ・コピーコンストラクタ・コピー代入演算子のうち1つでも定義したら、3つ全部定義せよ」**

```cpp
class Buffer {
    int* data;
public:
    ~Buffer() { delete[] data; }       // デストラクタだけ書いた
    // ↓ コンパイラがコピーを自動生成 → 浅いコピー → 二重解放バグ！
};
```

リソースを管理するクラスで1つだけ書くと、他のデフォルト生成と整合性が取れず破綻する。

### Rule of Five（C++11〜）

ムーブが加わって5つになる：

1. デストラクタ
2. コピーコンストラクタ
3. コピー代入演算子
4. ムーブコンストラクタ
5. ムーブ代入演算子

### Rule of Zero（モダンC++の結論）

**「何も書かないのが一番」**。STLに任せる。

```cpp
class Buffer {
    std::vector<int> data;   // vectorが自動で深いコピーしてくれる
    // デストラクタも・コピーも・ムーブも書かなくていい
};
```

自作で生ポインターを管理するのではなく、`std::vector` / `std::string` / `std::unique_ptr` を使えば全部自動で正しく動く。

### `const T&` で受ける理由

```cpp
Foo(const Foo& other);    // ✅ 正解
Foo(Foo other);           // ❌ 値渡しだと「コピーするためのコピー」が必要で無限再帰
Foo(Foo& other);          // ❌ const rvalue や const 変数が渡せない
```

### 自己代入チェック

```cpp
Foo& operator=(const Foo& o) {
    if (this == &o) return *this;   // 自分自身への代入を防ぐ
    delete[] data;
    // ...
}
```

`a = a` のコードで、`delete` してから自分をコピーしようとして壊れるのを防ぐ。

### コピーが呼ばれる意外な場面

```cpp
void func(Foo f);                       // ← 値渡しでコピー発生
Foo get() { Foo f; return f; }           // ← 戻り値でコピー発生（elision可）
std::vector<Foo> v;
v.push_back(Foo());                      // ← コピー発生（or ムーブ）
```

普通に書いているだけで知らぬ間にコピーされているのが C++ の特徴。

### 参照的な動作が欲しい時の5つの選択肢

| 方法 | 書き方 | 用途 |
|---|---|---|
| 参照 `T&` | `int& r = a;` | 関数引数、別名付け |
| const参照 `const T&` | `const int& r = a;` | 読み取り専用・コピー回避 |
| 生ポインター `T*` | `int* p = &a;` | 付け替え可能、nullptr可 |
| `std::shared_ptr<T>` | 共有所有 | **C#の参照に最も近い** |
| `std::reference_wrapper<T>` | コンテナに参照を入れたい時 | 特殊用途 |

### 関数引数の使い分け

```cpp
// 読み取り専用（コピー回避）→ const 参照（★最頻出）
void print(const std::vector<int>& v);

// 書き換えたい → 参照
void sort_in_place(std::vector<int>& v);

// オプショナル（nullableなもの）→ ポインター
void maybe_log(const Logger* logger);

// 所有権を奪いたい → ムーブ or unique_ptr
void take(std::unique_ptr<Foo> p);
```

### ポインターは例外的にアドレスの値コピー

```cpp
int* a = new int(10);
int* b = new int(20);
b = a;          // アドレスのコピー、両方が同じintを指す
*b = 100;       // *a も 100 になる
```

**ポインターの「値」はアドレス**なので、実質的に「参照コピー」のように振る舞う。C#経験者はこれを足がかりに理解するとOK。

## C#との比較

| 項目 | C# | C++ |
|---|---|---|
| `=` のデフォルト | 参照コピー | **値コピー（中身も）** |
| 参照型のコピー | 参照だけ | **全要素コピー** |
| shallow/deep の区別 | あまり意識しない | **常に意識** |
| コピー方法を定義 | `ICloneable.Clone()` | **コピーコンストラクタ** |
| コピー代入 | 新しい参照を代入 | **operator= を書く** |
| Rule of Three/Five | 不要（GCがある） | **必須**（生リソース管理時） |
| 参照セマンティクス | デフォルト | `shared_ptr` / `&` で実現 |
| 値型 | `struct` のみ | **クラスも値型が基本** |
| 関数引数のコピー | 参照型は参照渡し | **値渡しでコピー発生** |
| null参照 | `null`（参照型） | `nullptr`（ポインター） |

## 補足

### 「参照セマンティクスが欲しい時」の推奨

**初学者は `shared_ptr` が安全**。C#のGCと同じ「最後の一人がいなくなれば自動解放」。

```cpp
auto v = std::make_shared<std::vector<int>>(
    std::initializer_list<int>{1, 2, 3}
);
auto v2 = v;   // 参照カウント2、同じvector
v2->push_back(4);
// v も v2 も {1,2,3,4}
```

慣れたら使い分け：
- 所有権明確 → `unique_ptr` / 値型
- 借りるだけ → `&` / `const&`
- 共有 → `shared_ptr`

### コピー代入の例外安全：Copy-and-swap idiom

自己代入チェック不要で例外安全な書き方：

```cpp
Foo& operator=(Foo other) {   // 値渡しで受ける（コピー or ムーブ）
    swap(*this, other);        // 中身を入れ替え
    return *this;
}   // other のデストラクタが古い中身を解放
```

- 値渡しで受けるので自己代入時も安全
- 例外が投げられても元の状態が保たれる
- やや高度な技法

### 参照を返すのは危険

```cpp
int& danger() {
    int x = 10;
    return x;        // ❌ ローカル変数への参照！
}

int& y = danger();   // ゴミを指す（ダングリング参照）
```

参照・ポインターは「他に生きているもの」を指す必要がある。ローカル変数を返すのは禁忌。

### 値コピーが重い場合の最適化の方向性

```cpp
std::vector<int> getBig() {
    std::vector<int> v(1000000);
    return v;                       // C#感覚だと参照返しだが…
}

auto x = getBig();   // 本来100万要素コピーだが、実際は最適化される
```

- **RVO（Return Value Optimization）**：コピー省略
- **ムーブセマンティクス**：所有権の移動で高速化
- **const参照渡し**：引数のコピー回避

### コンストラクタの呼び出し種別

```cpp
Foo a;           // デフォルトコンストラクタ
Foo b = 5;       // 変換コンストラクタ（intから）
Foo c = a;       // コピーコンストラクタ
Foo d(a);        // コピーコンストラクタ（直接初期化）
Foo e{a};        // コピーコンストラクタ（リスト初期化）
Foo f;
f = a;           // コピー代入（コンストラクタではなく operator=）
```

### コピーの省略（Copy Elision）

```cpp
Foo get() {
    return Foo();   // 一時オブジェクトを返す
}

Foo f = get();   // 本来コピーコンストラクタ発動、でも現代コンパイラは省略
```

C++17 以降は特定条件でコピー省略が**必須**化された（NRVO/RVO）。目に見えないコピーは減っている。

### ポインターメンバーを持つクラスの最終解

**可能な限り `std::unique_ptr` / `std::shared_ptr` / `std::vector` に置き換える**。

```cpp
// ❌ 生ポインター（Rule of Three必要）
class Buffer {
    int* data;
    ~Buffer() { delete[] data; }
    // コピー・代入も書く必要あり
};

// ✅ unique_ptr（Rule of Zero）
class Buffer {
    std::unique_ptr<int[]> data;
    // 何も書かなくて良い（コピー不可になるが、通常はそれでOK）
};

// ✅ vector（Rule of Zero、コピー可）
class Buffer {
    std::vector<int> data;
};
```
