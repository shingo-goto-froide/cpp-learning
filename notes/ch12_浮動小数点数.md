# 第12章: 浮動小数点数

## 要点

### 浮動小数点数型の種類

| 型 | サイズ | 有効桁数 |
|---|---|---|
| `float` | 4バイト | 約7桁 |
| `double` | 8バイト | 約15桁 |
| `long double` | 環境依存 | 約18桁以上 |

- リテラルはデフォルトで`double`

```cpp
auto a = 1.0;    // double
auto b = 1.0f;   // float
auto c = 1.0L;   // long double
```

### 有効桁数

- 浮動小数点数は内部的に2進数で表現するため、10進数の値を正確に保持できないことがある

```cpp
double a = 0.1 + 0.2;
// 0.30000000000000004（0.3にならない）
```

- `float`は約7桁、`double`は約15桁までしか正確に表現できない

```cpp
float f = 123456789.0f;   // 123456792.0 になる（8桁目以降が不正確）
double d = 123456789.0;   // 正確に保持できる
```

### 浮動小数点数の比較

- `==`での比較は誤差があるため危険

```cpp
double a = 0.1 + 0.2;
a == 0.3;  // false（誤差のため）
```

- 許容誤差（イプシロン）を使って比較する

```cpp
#include <cmath>
bool nearly_equal(double a, double b) {
    return std::abs(a - b) < 1e-10;
}
```

### NaN（Not a Number）

- 「数学的に答えが定義できない計算」の結果

```cpp
0.0 / 0.0          // NaN（0÷0は定義されない）
std::sqrt(-1.0)    // NaN（負の数の平方根は実数にない）
1.0 / 0.0          // Infinity（無限大 — NaNではない）
```

- NaNは**自分自身と等しくない**（「値がわからない」ので比較できない）

```cpp
auto x = 0.0 / 0.0;
x == x;  // false
```

- NaN判定には`std::isnan()`を使う

```cpp
#include <cmath>
if (std::isnan(x)) { /* xはNaN */ }
```

- NaNを含む計算は結果もすべてNaNに伝播する

```cpp
auto x = 0.0 / 0.0;  // NaN
auto y = x + 1.0;     // NaN
auto z = y * 100.0;   // NaN
// 最終結果をチェックすれば途中の異常に気づける
```

### 浮動小数点数同士の変換

- `double` → `float`は精度が落ちる（暗黙的に変換されるが警告が出ることがある）

```cpp
double d = 1.23456789012345;
float f = d;  // 1.234568（約7桁に丸められる）
```

- `float` → `double`は精度が上がるが、元の誤差はそのまま

```cpp
float f = 0.1f;       // 0.1の近似値（float精度）
double d = f;          // float精度の誤差を引き継ぐ
```

### 整数型と浮動小数点数型の変換

- 浮動小数点数 → 整数は小数部が切り捨てられる

```cpp
double d = 3.9;
int i = d;  // 3（切り捨て）
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| デフォルトの浮動小数点型 | `double` | `double` |
| float指定 | `1.0f` | `1.0f` |
| NaN判定 | `double.IsNaN(x)` | `std::isnan(x)` |
| 0.1 + 0.2の誤差 | 同じ（IEEE 754） | 同じ（IEEE 754） |
| 精度を求める場合 | `decimal`型がある | 標準には`decimal`相当がない |

## 補足

### なぜ0.1 + 0.2が0.3にならないのか

- 0.1は2進数では無限小数（`0.0001100110011...`）になるため、有限ビットでは正確に表現できない
- これはC++に限らず、IEEE 754を使うすべての言語（C#、JavaScript等）で同じ

### お金の計算に浮動小数点数を使ってはいけない

- C#では`decimal`型を使うのが定番
- C++には`decimal`相当がないため、整数で「円」や「銭」単位で計算するか、外部ライブラリを使う
