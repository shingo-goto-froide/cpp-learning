# 第7章: 入出力

## 要点

### std::cin による入力

- `>>` 演算子で空白区切りの値を1つずつ読む
- 複数回呼ぶと順番に読んでいく

```cpp
double height{};
std::cout << "height(m)>";
std::cin >> height;

double mass{};
std::cout << "mass(kg)>";
std::cin >> mass;
```

- `std::cin >> x` は**空白・改行で区切られた1単語**だけ読む

```cpp
// 入力: hello world
std::string x{};
std::cin >> x;
std::cout << x;  // 出力: hello（worldは読まれない）
```

### std::getline で1行まるごと読む

- 空白を含む1行全体を読みたい場合は `std::getline` を使う

```cpp
std::string line{};
std::getline(std::cin, line);
std::cout << line;  // 出力: hello world
```

### 組み込み型と標準ライブラリの型

- `bool`, `int`, `double`, `char` などは**組み込み型**（キーワード）→ `std::` 不要
- `std::string`, `std::cout`, `std::vector` などは**標準ライブラリ**で定義 → `std::` 必要

```cpp
bool b{};           // OK: 組み込み型
int x{};            // OK: 組み込み型
std::string s{};    // OK: 標準ライブラリ
// std::bool b{};   // エラー: std::boolは存在しない
```

### {} による初期化の重要性

- 組み込み型では `{}` の有無で初期値が変わる
- `std::string` はデフォルトコンストラクタがあるのでどちらも空文字列

```cpp
int x{};    // 0で初期化（安全）
int x;      // 未初期化（値は不定、危険）

std::string s{};  // 空文字列
std::string s;    // これも空文字列（どちらでも同じ）
```

### リダイレクト（シェルの機能）

- プログラムのコードを変更せずに、入出力先をファイルに切り替えられる

```bash
# 標準入力をファイルから読む
./main.exe < input.txt

# 標準出力をファイルに書く
./main.exe > output.txt

# 両方
./main.exe < input.txt > output.txt
```

- **PowerShellでは `<` リダイレクトが使えない**（Bashやcmdでは使える）

```powershell
# PowerShellでの代替方法
Get-Content input.txt | ./main.exe
cmd /c "./main.exe < input.txt"
```

### パイプ

- プログラムの出力を別のプログラムの入力につなげる

```bash
# lsの出力をファイルに保存（リダイレクト）
ls > files.txt

# lsの出力をプログラムの入力に渡す（パイプ）
ls | ./program
```

- **リダイレクト**: プログラムとファイルをつなぐ
- **パイプ**: プログラムとプログラムをつなぐ

### ラムダ式（本の書き方について）

- 本ではmain()の中にラムダ関数を書くスタイルが多い
- これは教材の都合（関数やクラスを未説明の段階でコードの再利用を見せるため）
- 実務ではmain()にラムダを並べるスタイルは一般的ではない

```cpp
// STLアルゴリズムに渡す（実務でよく使う場面）
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;
});
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 1行読む | `Console.ReadLine()` | `std::getline(std::cin, x)` |
| 空白区切り入力 | `ReadLine()` + `Split()` | `std::cin >> x`（自動で分割） |
| 出力 | `Console.WriteLine()` | `std::cout << x << "\n"` |
| 未初期化変数 | コンパイルエラー | コンパイルは通る（値は不定） |
| ラムダ構文 | `(x) => x * 2` | `[](int x){ return x * 2; }` |
| キャプチャ | 自動 | `[]` で明示指定 |

## 補足

### Bash と PowerShell の違い

- C++のツール（g++, makeなど）はLinux由来なのでBashとの相性が良い
- ネットのC++関連の例もほぼBash前提

| 操作 | Bash | PowerShell |
|------|------|------------|
| リダイレクト入力 `<` | 使える | 使えない |
| パイプの中身 | テキスト | オブジェクト |
| 環境変数 | `$HOME` | `$env:USERPROFILE` |

- VSCodeのデフォルトターミナルをBashに変更可能: `Ctrl+Shift+P` → `Terminal: Select Default Profile` → `Git Bash`

### ビルドと実行

- PowerShellでは `make` が使えない → `mingw32-make` を使う
- VSCodeでは `Ctrl+Shift+B` でビルド＆実行が可能（tasks.json設定済み）
