# 第20章: コピー

## 要点

### 添字演算子のオーバーロード

- `operator[]` を定義すると、オブジェクトに `[]` でアクセスできる
- 参照を返すことで読み書き両方に対応できる

```cpp
struct MyArray {
    int data[5] = {10, 20, 30, 40, 50};

    int & operator [](std::size_t i) {
        return data[i];
    }
};

MyArray a;
int x = a[2];   // 読み取り → 30
a[2] = 99;      // 書き込みもできる（参照を返しているため）
```

| 部分 | 意味 |
|---|---|
| `int &` | 戻り値は `int` の参照（代入もできる） |
| `operator []` | `[]` 演算子を定義する |
| `std::size_t i` | 引数はインデックス |

### const版の添字演算子

- 読み取り専用にしたい場合は `const` 版を定義する

```cpp
struct MyArray {
    int data[5];

    // 読み書き用
    int & operator [](std::size_t i) {
        return data[i];
    }

    // 読み取り専用（constオブジェクト用）
    const int & operator [](std::size_t i) const {
        return data[i];
    }
};
```

### 参照の `&` の位置

- 型の前でも後でも意味は同じ。スタイルの違いだけ

```cpp
int& x;   // 型に寄せる
int &x;   // 変数名に寄せる
int & x;  // 間にスペース
// すべて同じ意味
```

- ただし複数変数を1行で宣言すると罠がある

```cpp
int& a, b;  // a は参照、b はただの int！
```

- `&` は変数ごとにかかるため、型に寄せると誤解しやすい
- 実務では **1行に1変数** が普通なのでどちらでも問題にならない

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 添字アクセスの定義 | `this[int i]`（インデクサー） | `operator [](std::size_t i)` |
| 読み書きの制御 | get / set を別々に定義 | 参照を返せば両方まかなえる |
| 読み取り専用 | `get` のみ定義 | `const` 版を定義 |
| 参照の記号 | `ref` キーワード | `&` 記号（位置は自由） |
| 複数変数宣言 | `int a, b;` は両方 `int` | `int& a, b;` は `a` だけ参照 |

## 補足

### `&` の位置のスタイル派閥

| スタイル | 好まれる場面 |
|---|---|
| `int& x`（型寄せ） | C# / Java 出身者に自然。「型が参照」と読める |
| `int &x`（変数名寄せ） | C/C++ の伝統的スタイル。江添本もこちら寄り |

- どちらも正しいが、プロジェクト内では統一するのが大事
