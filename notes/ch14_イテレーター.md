# 第14章: イテレーター

## 要点

### イテレーターとは

- コンテナの要素を順番にたどるための仕組み
- C#の`IEnumerator`に相当する

```cpp
std::vector<int> v = {1, 2, 3};

auto iter = std::begin(v); // 先頭を指すイテレーター
*iter;   // 1（値を取り出す）
++iter;  // 次の要素に進む
*iter;   // 2
```

### *（間接参照）

- `*`はイテレーターが指している値を取り出す操作
- ポインタの間接参照と同じ記法

```cpp
auto it = std::begin(v);
*it; // 先頭要素の値
```

### begin() と end()

- `std::begin(v)` — 先頭要素を指すイテレーター
- `std::end(v)` — **最後の要素の1つ後ろ**を指す（要素ではない）

```
要素:  [ 1 ][ 2 ][ 3 ][ 4 ][ 5 ][   ]
        ↑                          ↑
      begin()                    end()
```

### 先頭から最後まで出力

```cpp
std::vector<int> v = {1, 2, 3, 4, 5};

for (auto i = std::begin(v); i != std::end(v); ++i) {
    std::cout << *i << " "s;
}
// 1 2 3 4 5
```

- `i != std::end(v)` で「まだ終わりじゃない間」ループする

### at(i) とイテレーターの使い分け

| | at(i) | イテレーター |
|---|---|---|
| 特定の位置 | 得意 | 回りくどい |
| 順番にたどる | できる | 得意 |
| 範囲指定 | できない | できる |
| 標準アルゴリズム | 使えない | 必須 |
| list等への対応 | 不可 | 可能 |

- 自分でループするなら`at()`や範囲for文で十分
- 標準ライブラリの機能を使うならイテレーターが必須

### 標準アルゴリズムとの連携

- `std::sort`や`std::find`などはイテレーターを受け取る

```cpp
std::vector<int> v = {3, 1, 2};

std::sort(std::begin(v), std::end(v));

auto it = std::find(std::begin(v), std::end(v), 2);
```

### 範囲を指定できる

- イテレーターを使えば「ここからここまで」を指定できる

```cpp
std::vector<int> v = {5, 3, 1, 4, 2};

// 先頭3つだけソート
std::sort(std::begin(v), std::begin(v) + 3);
// {1, 3, 5, 4, 2}
```

### どのコンテナでも同じ書き方

- `vector`でも`list`でも同じイテレーターの書き方で扱える
- `list`には`at()`がないので、イテレーターでしかアクセスできない

```cpp
std::vector<int> v = {1, 2, 3};
std::list<int> l = {1, 2, 3};

// どちらも同じ書き方
for (auto i = std::begin(v); i != std::end(v); ++i) { }
for (auto i = std::begin(l); i != std::end(l); ++i) { }
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| イテレーター | `IEnumerator` | イテレーター（ポインタ風） |
| 値の取得 | `iter.Current` | `*iter` |
| 次に進む | `iter.MoveNext()` | `++iter` |
| 終了判定 | `MoveNext()`が`false` | `iter != end()` |
| 直接触る頻度 | 少ない（`foreach`が隠蔽） | 標準アルゴリズム利用時は必須 |
| 範囲指定 | LINQの`Skip`/`Take` | `begin() + n`で直接指定 |

## 補足

### 範囲for文とイテレーターの関係

- 範囲for文は内部的にイテレーターを使っている

```cpp
// この範囲for文は…
for (auto x : v) { std::cout << x; }

// これと同じ意味
for (auto it = std::begin(v); it != std::end(v); ++it) {
    auto x = *it;
    std::cout << x;
}
```

### end()が「1つ後ろ」を指す理由

- 空のコンテナでも`begin() == end()`で判定できる
- ループの終了条件が`!=`で統一的に書ける
- 範囲を`[begin, end)`の半開区間で表す慣習（数学と同じ）
