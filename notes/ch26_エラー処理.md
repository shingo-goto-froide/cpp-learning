# 第26章: エラー処理

## 要点

### at() による範囲チェック

- `operator[]` はチェックなし、`at()` は範囲外で例外を投げる

```cpp
std::array<int, 5> a = {1, 2, 3, 4, 5};

a[10];    // チェックなし（未定義動作）
a.at(10); // 範囲外 → std::out_of_range 例外
```

- `at()` の自作実装：

```cpp
reference at(std::size_t i) {
    if (i >= N) {
        throw std::out_of_range("index out of range");
    }
    return storage[i];
}
```

- `at()` が `[]` より遅いと言われるのは `if` 文があるから。ただし現代のCPUでは最適化されてほぼ差がないことが多い

### 例外クラス

- `std::out_of_range` のコンストラクタはエラーメッセージを受け取る

```cpp
// 宣言 — 引数名は省略できる（型だけでOK）
out_of_range(std::string const &);

// 引数名ありでも同じ
out_of_range(std::string const & message);
```

- `what()` でエラーメッセージを取り出せる

```cpp
try {
    a.at(10);
} catch (std::out_of_range & e) {
    std::cout << e.what();  // → "index out of range"
}
```

### 宣言では引数名を省略できる

- コンパイラは**型さえわかれば関数を区別できる**ので、宣言では引数名は不要

```cpp
// どちらも同じ宣言
void print(std::string const &);
void print(std::string const & message);

// 定義では引数を使うので名前が必要
void print(std::string const & message) {
    std::cout << message;
}
```

### try/catch

- `catch` があれば例外が発生してもプログラムは止まらず続行できる
- `catch` がどこにもなければプログラムは強制終了する

```cpp
try {
    throw std::runtime_error("error");
} catch (std::runtime_error & e) {
    std::cout << "エラー処理";
}
std::cout << "プログラム続く";  // 続行される
```

### スタックアンワインディング

- 例外が投げられたとき、**呼び出し元を順番に遡りながら途中の変数を自動的に破棄していく**仕組み

```cpp
void c() {
    std::vector<int> v = {1, 2, 3};  // ヒープにメモリ確保
    throw std::runtime_error("error");
    // v のデストラクタが自動で呼ばれる → メモリ解放
}

void b() {
    std::string s = "hello";
    c();
    // s のデストラクタが自動で呼ばれる
}

void a() {
    try {
        b();
    } catch (std::runtime_error & e) {
        // ここでキャッチ
    }
}
```

- `catch` して続行するとき、もしデストラクタが呼ばれなかったらメモリリークが積み重なる
- プログラムがすぐ終了するなら OS が回収するので実害は少ない
- **長時間動くプログラム**（サーバー、ゲーム等）では重要

### デストラクタが呼ばれるタイミング

- **スコープを抜ける方法が何であっても呼ばれる**

| 抜け方 | デストラクタ |
|---|---|
| `return` | 呼ばれる |
| 関数の `}` に到達 | 呼ばれる |
| 例外（スタックアンワインディング） | 呼ばれる |

### std::scope_exit

- スコープを抜けるときに**指定した処理を自動で実行する**仕組み
- C++には `finally` がないので、代わりにこれを使う

```cpp
#include <scope>

void func() {
    auto e = std::scope_exit([]{ std::cout << "後片付け"; });

    // どんな抜け方をしても「後片付け」が実行される
}
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 範囲チェック | `[]` で自動チェック | `[]` はチェックなし、`at()` で明示的にチェック |
| エラーメッセージ取得 | `e.Message` | `e.what()` |
| 例外クラス | `ArgumentOutOfRangeException` | `std::out_of_range` |
| 後片付け | `finally` ブロック | `std::scope_exit`（`finally` はない） |
| メモリ回収 | ガベージコレクタがあるので意識不要 | デストラクタ + スタックアンワインディングで回収 |
| 例外の方針 | 基本的に例外を使う | 現場によって異なる（使わない現場もある） |

## 補足

### C++の例外に対する方針の違い

| 方針 | 使う現場 |
|---|---|
| 例外を使う | 一般的なアプリケーション |
| 例外を使わない | ゲーム、組み込み、Google（パフォーマンス理由） |

- C#では `try/catch` が当たり前だが、C++では戻り値でエラーを返すスタイルも普通にある

## 理解度

- [x] `at()` は範囲チェック付き、`[]` はチェックなし
- [x] `at()` が遅いのは `if` 文があるから（実際はほぼ差がない）
- [x] 宣言では引数名を省略できる（型だけでOK）
- [x] `what()` でエラーメッセージを取り出せる
- [x] `catch` があれば続行、なければプログラム終了
- [x] スタックアンワインディングで途中の変数が自動破棄される
- [x] デストラクタはスコープを抜ける方法に関係なく呼ばれる
- [x] `std::scope_exit` は C# の `finally` に相当する
