# 第11章: 整数

## 要点

### 2進数リテラル（0b / 0B）

- プレフィックス`0b`または`0B`で2進数を記述できる

```cpp
auto a = 0b1010;   // 10
auto b = 0B1111;   // 15
```

- リテラルの型はデフォルトで`int`（10進数リテラルと同じ扱い）
- サフィックスで型を変更できる

```cpp
auto a = 0b1010;    // int
auto b = 0b1010u;   // unsigned int
auto c = 0b1010L;   // long
auto d = 0b1010LL;  // long long
```

- `int`（4バイト）なら32ビット分、`long long`（8バイト）なら64ビット分まで記述可能
- 長いビット列は16進数の方が読みやすい

```cpp
// 32ビット全部2進数で書くと長い
auto a = 0b1111'1111'0000'0000'1111'1111'0000'0000;

// 16進数なら短い
auto b = 0xFF00FF00;
```

### 数値の桁区切り（'）

- `'`を任意の位置に挿入して数値を読みやすくできる

```cpp
auto a = 0b1111'0000;  // 2進数を4桁ずつ
auto b = 1'000'000;    // 10進数を3桁ずつ
```

- 区切りの位置は自由（意味のある位置に入れる）

### ビット操作の活用例

- フラグ管理にビット操作を使うと、1つの整数で複数の状態を管理できる

```cpp
auto state     = 0b0000'0000;
auto poisoned  = 0b0000'0001; // 毒
auto paralyzed = 0b0000'0010; // 麻痺
auto burning   = 0b0000'0100; // 炎上

state |= poisoned | burning;     // 毒+炎上をセット
state &= ~poisoned;              // 毒を解除
if (state & burning) { /* ... */ } // 炎上中か判定
```

- `|=`（OR代入）は既存のビットを壊さずに新しいフラグを追加できる

```
  0000'0010  (state: 麻痺)
| 0000'0101  (poisoned | burning)
-----------
  0000'0111  ← 麻痺はそのまま、毒と炎上が追加
```

- ゲーム開発では衝突マスク、入力の同時押し判定、タイルマップの属性管理などで使われる
- `bool`配列より省メモリで、比較が速く、ネットワーク通信やセーブデータにも有利

### 符号付き整数型の種類

| 型 | 最小サイズ |
|---|---|
| `short` | 2バイト |
| `int` | 2バイト |
| `long` | 4バイト |
| `long long` | 8バイト |

- サイズは**最小保証**であり、環境によって異なる可能性がある
- 現代の主要環境（Windows/Mac/Linux）では`int`は4バイト

### 固定幅整数型（<cstdint>）

- サイズを厳密に固定したいときに使う

```cpp
#include <cstdint>

std::int16_t a; // 必ず2バイト
std::int32_t b; // 必ず4バイト
std::int64_t c; // 必ず8バイト
```

### 整数型の範囲

- 符号付き32ビットの場合、1ビットは符号に使うので値に使えるのは31ビット
- `2^31 = 2,147,483,648` → 正の最大値は `2^31 - 1`（0から数えるため）

| 型 | バイト数 | 範囲 | ざっくり |
|---|---|---|---|
| `short` | 2 | -32,768 ~ 32,767 | 約3万 |
| `int` | 4 | -2,147,483,648 ~ 2,147,483,647 | 約21億 |
| `long long` | 8 | -9,223,372,036,854,775,808 ~ ... | 約922京 |

### サイズと範囲の取得

```cpp
// サイズ（バイト数）
sizeof(int)       // 4
sizeof(long long) // 8

// 範囲
#include <limits>
std::numeric_limits<int>::min() // -2,147,483,648
std::numeric_limits<int>::max() //  2,147,483,647
```

### char型

- 整数型だが**文字として扱われる**特殊な型（1バイト）

```cpp
char c = 'A';
std::cout << c;     // A（文字として出力）
std::cout << c + 0; // 65（整数として出力）
```

- 内部的にはASCIIコードの数値が入っているだけ

```cpp
char c = 65;  // 'A'と同じ
```

- 文字列（`std::string`）の中身は`char`の配列なので、1文字単位の操作 = `char`を触ること

```cpp
std::string s = "hello";
s[0] = 'H';  // "Hello"
```

- ASCIIコードの連番を利用した変換が可能

```cpp
char lower = 'A' + 32;    // 'a'（大文字→小文字）
char digit = '3' - '0';   // 3（文字→数値）
```

### 範囲for文（range-based for）

- C#の`foreach`と同じ

```cpp
// C++
std::string s = "Hello";
for (char c : s) {
    // sの各文字がcに入る
}

// C# の foreach (char c in s) と同じ
```

- `vector`にも使える

```cpp
std::vector<int> v = {1, 2, 3};
for (int x : v) {
    std::cout << x << "\n";
}
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 2進数リテラル | `0b`（同じ） | `0b` / `0B` |
| 桁区切り | `_` | `'` |
| intのサイズ | 必ず4バイト | 最小2バイト（環境依存） |
| 固定幅整数 | `short`/`int`/`long`（固定） | `std::int16_t`/`int32_t`/`int64_t` |
| フラグ管理 | `[Flags] enum`が定番 | 整数で直接ビット操作が多い |
| 範囲の取得 | `int.MinValue`/`int.MaxValue` | `std::numeric_limits<int>::min()`/`max()` |
| charのサイズ | 2バイト（UTF-16） | 1バイト（ASCII） |
| foreach | `foreach (var x in s)` | `for (auto x : s)` |
| stringの変更 | 不変（直接変更不可） | 可変（`s[0] = 'H'`可能） |

## 補足

### C++の整数型サイズが固定でない理由

- C++はハードウェアに近い言語であり、多様なプラットフォーム（組み込み、デスクトップ等）で動作する
- そのためサイズを「最小保証」にして環境ごとの最適なサイズを許容している
- サイズが重要な場面（ファイルフォーマット、ネットワーク通信等）では`<cstdint>`の固定幅型を使う

### ビットフラグ vs bool配列

| | ビットフラグ | bool配列 |
|---|---|---|
| メモリ | 32個で4バイト | 32個で32バイト |
| 比較 | 一発で全比較 | ループが必要 |
| 保存/通信 | 整数1つ送ればOK | 配列を送る |
