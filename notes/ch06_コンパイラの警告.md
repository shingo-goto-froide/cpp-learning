# 第6章: コンパイラの警告

## 要点

### -Wunused-variable

- 宣言したのに使っていない変数を警告する
- `-Wall` に含まれているため、個別指定は不要

```cpp
int main() {
    int x = 10;  // 警告: unused variable 'x'
}
```

### -Wall に含まれる主な警告

| オプション | 内容 |
|---|---|
| `-Wunused-variable` | 未使用の変数 |
| `-Wunused-function` | 未使用の関数 |
| `-Wuninitialized` | 未初期化の変数の使用 |
| `-Wswitch` | switchで列挙値の漏れ |
| `-Wparentheses` | 条件式内の代入（`if (x = 0)`） |

### 条件式内の代入（`=` と `==` の間違い）

- C++では `if (x = 0)` が書けてしまう（代入してからその値で判定）
- `-Wall`（`-Wparentheses`）で警告が出る

```cpp
if (x = 0)    // 警告あり: 代入ミスの可能性
if ((x = 0))  // 警告なし: 括弧で囲むと「意図的」とみなされる
if (x == 0)   // 正しい比較
```

### 括弧の意味

- `if ((x = 0))` のように括弧で囲むと「この代入は意図的です」というコンパイラへの合図になる
- 警告が抑制されるため、本当に意図的な場合のみ使うべき

```cpp
// 意図的な代入の実用例
std::string line;
while ((std::getline(std::cin, line))) {
    std::cout << line;
}
```

### 防ぎ方

1. **ヨーダ記法**: `if (0 == x)` — 書き間違えるとコンパイルエラーになる（可読性は下がる）
2. **`-Werror=parentheses`**: 警告をエラーに昇格させる
3. **代入と条件を分離する**: 最も安全で読みやすい

```cpp
x = compute();
if (x) { /* ... */ }
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 未使用変数の警告 | IDEがリアルタイム表示 | `make`するまで気づけない |
| 条件式内の代入 | コンパイルエラー | 書ける（`-Wall`で警告） |
| 警告の厳格さ | デフォルトで厳格 | オプション（`-Wall`等）で制御 |

## 補足

- C#では条件式内の代入が言語仕様で禁止されているため、`=`と`==`の取り違えは起きない
- C++では `-Wall` を付けていれば大半の危険なコードに警告が出るので、常に有効にしておくことが重要

- [x] 括弧による警告抑制の仕組み
- [x] ヨーダ記法などの防止策
