# 第2章: 開発環境構築

## 要点

### MinGW-w64

- Windows 上で GCC（GNU Compiler Collection）を使うためのツールセット
- インストール後、`C:\mingw64\bin\` を環境変数 PATH に追加する

### コンパイルオプション

| オプション | 説明 |
|------------|------|
| `-std=c++20` 等 | 使用する C++ 規格を指定 |
| `-Wall` | 主要な警告をすべて有効にする |
| `-pedantic-errors` | 規格に厳密に従わないコードをエラーにする |

- `-Wall` と `-pedantic-errors` は**常につけるべき**。なしだと問題のあるコードが通ってしまう

### プリコンパイル済みヘッダー (.gch)

- ヘッダーファイルを事前にコンパイルしておく仕組み
- `all.hpp` → `all.hpp.gch` にコンパイルすることでビルド時間を短縮
- コンパイラがヘッダーと同名の `.gch` を自動検出して使う

### Makefile

```makefile
program : main.cpp all.hpp.gch
	g++ -std=c++17 -Wall -pedantic-errors -include all.hpp main.cpp -o program

all.hpp.gch : all.hpp
	g++ -std=c++17 -Wall -pedantic-errors all.hpp -o all.hpp.gch

run : program
	./program

.PHONY : run clean

clean :
	rm -f program all.hpp.gch
```

- **PHONY ターゲット**: 実際のファイルを生成しないターゲットに指定する（`run`, `clean` 等）
  - これがないと、同名のファイルが存在した場合に実行されない

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| ビルドツール | MSBuild / `dotnet build`（`.csproj` が設定） | `g++` コマンド直接 or `make` / CMake |
| プロジェクト設定 | `.csproj`（XML）に自動管理 | Makefile に手動で記述 |
| 警告レベル | `<WarningLevel>` / `<TreatWarningsAsErrors>` | `-Wall` / `-pedantic-errors` |
| パッケージ管理 | NuGet | なし（ヘッダーを直接配置 or vcpkg / Conan 等） |

- C# では `dotnet new` でプロジェクト雛形が作られるが、C++ はファイル構成を自分で決める
- Makefile は C# の `.csproj` に相当するが、ビルドルールを自分で書く必要がある

## 補足

- `make` は依存関係を見てファイルが更新された部分だけ再コンパイルする → 大規模プロジェクトで重要
- `-include all.hpp` オプションで、ソースコード内に `#include` を書かなくても自動でインクルードされる
