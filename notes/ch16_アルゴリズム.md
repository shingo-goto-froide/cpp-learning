# 第16章: アルゴリズム

## 要点

### std::for_each

- 各要素に関数を適用する
- 第3引数に関数（ラムダ式）を渡す

```cpp
std::vector<int> v = {1, 2, 3};

std::for_each(std::begin(v), std::end(v), [](auto value) {
    std::cout << value;
});
```

- ラムダを変数に入れると読みやすくなる

```cpp
auto print = [](auto value) { std::cout << value; };
std::for_each(std::begin(v), std::end(v), print);
```

- C++20以降は Ranges でコンテナを直接渡せる

```cpp
std::ranges::for_each(v, print);
```

- 単純なループなら範囲for文のほうが読みやすい

```cpp
for (auto value : v) {
    std::cout << value;
}
```

### ラムダ式のキャプチャ

- `[]` の中で外の変数を使うかどうか・どう取り込むかを指定する

```cpp
int x = 10;
int y = 20;

auto f1 = [=]() { return x + y; };  // 全部コピーでキャプチャ
auto f2 = [&]() { x += 1; };        // 全部参照でキャプチャ
auto f3 = [x]() { return x; };      // xだけコピー
auto f4 = [&x]() { x += 1; };       // xだけ参照
auto f5 = [=, &x]() { };            // 基本コピー、xだけ参照
auto f6 = []() { };                  // キャプチャなし（外の変数は使えない）
```

- キャプチャなし `[]` で外の変数を使うとコンパイルエラーになる
- C#ではキャプチャは自動（全部参照）だが、C++では明示的に指定が必要
- コピーか参照かでパフォーマンスや挙動が変わるため、意図しないキャプチャを防ぐ設計

### ラムダを変数に入れる

```cpp
// auto — 型はコンパイラが決める（高速）
auto print = [](auto value) { std::cout << value; };

// std::function — 型を明示（オーバーヘッドあり）
std::function<void(int)> print = [](int value) { std::cout << value; };
```

- ローカル変数なら `auto` が推奨
- `std::function` は内部で間接呼び出しのコストがかかる
- クラスのメンバや関数の引数の型として宣言するときは `std::function` を使う

### all_of / any_of / none_of

```cpp
std::vector<int> v = {2, 4, 6, 8};
auto is_even = [](auto x) { return x % 2 == 0; };

std::all_of(std::begin(v), std::end(v), is_even);   // true — 全部偶数か
std::any_of(std::begin(v), std::end(v), is_even);   // true — 1つでも偶数か
std::none_of(std::begin(v), std::end(v), is_even);  // false — 偶数が1つもないか
```

### find / find_if

```cpp
std::vector<int> v = {1, 2, 3, 4, 5};

// find — 値が一致するものを探す
auto it = std::find(std::begin(v), std::end(v), 3);

// find_if — 条件（predicate）に一致するものを探す
auto it = std::find_if(std::begin(v), std::end(v), [](auto x) { return x > 3; });
```

- 見つからなかった場合は `std::end(v)` を返す（nullではない）
- 見つかったかどうかは `it != std::end(v)` で判定する
- **pred** = predicate（述語）の略。true/false を返す関数のこと

### transform / copy

```cpp
std::vector<int> a = {1, 2, 3};
std::vector<int> b(3);

// copy — そのまま写す
std::copy(std::begin(a), std::end(a), std::begin(b));
// b: {1, 2, 3}

// transform — 関数を適用してから写す
std::transform(std::begin(a), std::end(a), std::begin(b), [](auto x) { return x * 2; });
// b: {2, 4, 6}
```

- 書き込み先は `std::begin(b)` で指定する（`b` ではなくイテレーター）
- 「ここから書き始めて」という先頭位置を渡す設計

### replace / fill / generate

```cpp
std::vector<int> v = {1, 2, 3, 2, 5};

// replace — 特定の値を置き換える
std::replace(std::begin(v), std::end(v), 2, 99);
// v: {1, 99, 3, 99, 5}

// fill — 全部同じ値で埋める
std::fill(std::begin(v), std::end(v), 0);
// v: {0, 0, 0, 0, 0}

// generate — 関数の戻り値で埋める
int n = 0;
std::generate(std::begin(v), std::end(v), [&]() { return n++; });
// v: {0, 1, 2, 3, 4}
```

### remove / remove_if

```cpp
std::vector<int> v = {1, 2, 3, 2, 5};

// remove — 値が一致するものを除去
auto it = std::remove(std::begin(v), std::end(v), 2);

// remove_if — 条件に一致するものを除去
auto it = std::remove_if(std::begin(v), std::end(v), [](auto x) { return x % 2 == 0; });
```

- **重要**: `remove` は実際には要素を削除しない。前に詰めるだけでサイズは変わらない

```
remove前: [1][2][3][2][5]
remove後: [1][3][5][?][?]
                   ↑     ↑
                   it   end(v) ← まだ5要素
```

- `it` から `end(v)` の間はゴミデータ（アクセスできるがエラーにならない）
- 本当に削除するには `erase` と組み合わせる

```cpp
v.erase(it, std::end(v));
// v: {1, 3, 5} — これで本当に3要素になる
```

## C#との比較

| C++ | C# |
|------|------|
| `std::for_each(begin, end, func)` | `list.ForEach(func)` |
| `std::all_of` / `any_of` / `none_of` | `.All()` / `.Any()` / なし |
| `std::find(begin, end, 値)` | `.First(x => x == 値)` |
| `std::find_if(begin, end, 条件)` | `.First(x => 条件)` |
| `std::transform` | `.Select(x => ...).ToList()` |
| `std::copy` | `.ToList()` や `new List(original)` |
| `std::replace` | なし（`Select`で代用） |
| `std::fill` | `Array.Fill()` |
| `std::generate` | `Enumerable.Range()` が近い |
| `std::remove` + `erase` | `list.RemoveAll()` |
| 見つからない → `end()` を返す | 見つからない → 例外 or default |
| `auto f = [](auto x) { ... };` | `Func<T> f = x => ...;` |
| `std::function<void(int)>` | `Action<int>` |
| `[&]` / `[=]` で明示キャプチャ | 自動キャプチャ（参照） |

- C#の `RemoveAll` は一発で消えるが、C++は `remove` + `erase` の2ステップ
- C#ではラムダのキャプチャは自動だが、C++では `[]` で明示的に指定する
- C#では `Func` / `Action` が必須だが、C++では `auto` で済む（パフォーマンスも良い）

## 補足

### removeがサイズを変えない理由

- `std::remove` はイテレーター範囲に対して動作する汎用アルゴリズム
- イテレーターだけではコンテナのサイズを変更できない
- サイズ変更はコンテナ自身の仕事（`erase`）

### std::end(v) の正体

- `std::end(v)` はイテレーター（場所を指すもの）であり、値ではない
- 最後の要素の1つ後ろの「場所」を指す目印
- `remove` 後もvectorのサイズが変わらないので `end(v)` の位置も変わらない

### オーバーヘッドとは

- 「余分にかかるコスト」（処理時間やメモリ）のこと
- `std::function` は内部で型消去をしており、呼び出しごとに間接呼び出しコストがかかる
- `auto` ならコンパイラが直接インライン展開できるのでオーバーヘッドがない
