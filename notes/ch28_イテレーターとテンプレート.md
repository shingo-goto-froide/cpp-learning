# 第28章: イテレーターとテンプレート

## 要点

### テンプレートの基本文法

```cpp
template<typename T>   // 型パラメータを1つ取る
struct Box {
    T value;
};

Box<int>    b1;   // T = int
Box<double> b2;   // T = double
```

- `template<...>` でテンプレート宣言
- `typename T` または `class T` で型パラメータ（どちらも同じ意味）
- 使用時は `Box<int>` のように山カッコで型を指定

### 複数パラメータ

```cpp
template<typename Key, typename Value>
struct Pair { Key first; Value second; };
```

### 非型テンプレートパラメータ（値を渡す）

```cpp
template<typename T, std::size_t N>
struct Array { T data[N]; };

Array<int, 5> a;   // std::array の仕組みと同じ
```

### 関数テンプレート

```cpp
template<typename T>
T max_value(T a, T b) { return a > b ? a : b; }

max_value(3, 5);              // T は int と推論される
max_value<double>(1.1, 2.2);  // 明示的に指定も可
```

### `typename` の2つの意味

**① テンプレートパラメータの宣言**

```cpp
template<typename T>   // T は型
template<class T>      // 同じ意味
```

**② 「これは型です」とコンパイラに教える（依存型の解決）**

```cpp
template<typename Array>
struct array_iterator {
    using reference = typename Array::reference;
    //                ^^^^^^^^
    // Array::reference が型か変数か、Array 実体化まで分からないので
    // typename で「型」と明示する必要がある
};
```

### ネストした型エイリアス

- STLコンテナは内部に型エイリアスを公開している

```cpp
// std::array の内部（概念）
template<typename T, std::size_t N>
struct array {
    using value_type       = T;
    using reference        = T&;
    using const_reference  = const T&;
    using pointer          = T*;
    using iterator         = /* 実装依存 */;
    using size_type        = std::size_t;
};
```

- これを外から使える

```cpp
std::array<int, 5>::reference   // int&
std::array<int, 5>::value_type  // int
```

- イテレーター側でコンテナの型情報を再利用できる

```cpp
template<typename Array>
struct array_iterator {
    using reference = typename Array::reference;
    reference operator*() const { return (*a)[i]; }
};
```

### `const` メンバー関数

- 末尾に付けた `const` は**メンバー関数自身**にかかる
- 「この関数は `this` を書き換えません」という宣言

```cpp
reference operator*() const { return a[i]; }
//                    ^^^^^
//                    末尾の const
```

- **`const` オブジェクトからでも呼び出せる**のが最大のメリット

```cpp
const Counter c;
c.get_const();   // ✅ const メンバー関数なら呼べる
c.get_normal();  // ❌ 非 const メンバー関数は呼べない
```

- STLでは読み取り・書き込みで2版用意するのが定石

```cpp
int const& operator[](std::size_t i) const { return data[i]; }  // 読み取り
int&       operator[](std::size_t i)       { return data[i]; }  // 書き込み
```

### 演算子オーバーロード（`operator--` を例に）

```cpp
iota_iterator& operator--() noexcept {
    --value;
    return *this;
}
```

| 部分 | 意味 |
|---|---|
| `iota_iterator&` | 戻り値の型（自分自身のリファレンス） |
| `operator--` | `--` 演算子の動作定義 |
| `()` | 引数なし → **前置**デクリメント |
| `noexcept` | 例外を投げない保証 |

### 前置 vs 後置

```cpp
// 前置 --it
iota_iterator& operator--()    { --value; return *this; }

// 後置 it--
iota_iterator  operator--(int) {
    iota_iterator tmp = *this;  // 古い値を保存
    --(*this);                  // 自分を減らす
    return tmp;                 // 古い値を返す
}
```

- 後置は**ダミーの `int` 引数**で前置と区別する（言語仕様）
- 戻り値型も違う：前置はリファレンス、後置は値（コピー）
- C#の `operator--` は前置・後置の区別なし

### `noexcept`

- 「この関数は例外を投げない」とコンパイラに宣言
- **メリット**: 最適化のヒント、`std::vector` のムーブ最適化に影響、意図の明示
- **違反するとどうなるか**: 実行時に `std::terminate` でプログラム強制終了
- イテレーターの基本操作（`++`, `--`, `*`）は慣例的に `noexcept`

### イテレーターカテゴリー（5段階）

```
入力/出力 → 前方 → 双方向 → ランダムアクセス → 連続(C++20)
```

| カテゴリー | できること | 代表例 |
|---|---|---|
| 入力 | 1回読み取り、`++` | `istream_iterator` |
| 出力 | 書き込み、`++` | `ostream_iterator`, `back_inserter` |
| 前方 | 何度でも読める、`++` | `forward_list` |
| 双方向 | `++`, `--` | `list`, `map`, `set` |
| ランダムアクセス | `+=n`, `-=n`, `[i]`, 距離計算 | `vector`, `array` |
| 連続 (C++20) | さらにメモリ連続 | `vector`, `array` |

- **右に行くほど上位互換**（ランダムアクセスは全部できる）
- **データ構造の物理的性質**が能力を決める

### 自作イテレーターでのカテゴリー宣言

```cpp
template<typename Array>
struct array_iterator {
    using iterator_category = std::random_access_iterator_tag;
    using value_type        = typename Array::value_type;
    using reference         = typename Array::reference;
    using difference_type   = std::ptrdiff_t;
    using pointer           = typename Array::pointer;
};
```

### イテレーターの移動

```cpp
// ランダムアクセス（vector, array）→ 一発ジャンプ可
auto it = v.begin() + 3;   // ✅ O(1)
v.begin()[3];              // ✅
it += 5;                   // ✅

// 双方向以下（list, map）→ ステップごとにしか動けない
auto it = l.begin() + 3;   // ❌ コンパイルエラー
std::advance(it, 3);       // ✅ ただし内部で++を3回（O(n)）
```

| 関数 | 戻り値 | 元のイテレーター |
|---|---|---|
| `std::advance(it, n)` | なし | 書き換える |
| `std::next(it, n)` | 進んだ新しい it | 変わらない |
| `std::prev(it, n)` | 戻った新しい it | 変わらない |

### 配列とポインター

- `int arr[5]` は**配列**（型は `int[5]`）
- ただし多くの場面で**先頭要素へのポインター `int*` に自動変換**される（array-to-pointer decay）

```cpp
int arr[5] = {1,2,3,4,5};

int* p = arr;          // arr は &arr[0] に自動変換
std::cout << arr;      // ポインターとして出力
func(arr);             // 関数に渡すとポインターになる
```

### 配列とポインターは別物である証拠

```cpp
int arr[5];
int* p = arr;

sizeof(arr);   // 20（int × 5）
sizeof(p);     // 8（ポインター1つ分）

arr = other;   // ❌ 配列は付け替え不可
p = other;     // ✅ ポインターは付け替え可
```

### 「配列へのポインター」vs「先頭要素へのポインター」

```cpp
int arr[5];

int*   p1 = arr;        // 先頭要素（int 1個）へのポインター
int (*p2)[5] = &arr;    // 配列全体（int 5個）へのポインター

sizeof(*p1);   // 4（int 1個分）
sizeof(*p2);   // 20（配列全体）
```

- 日常会話で「配列を指すポインター」と言えばほぼ **`int*`**（先頭要素）のこと
- 真の「配列へのポインター」`int(*)[5]` は実務ではほぼ出てこない

### `p[i]` の正体

```cpp
p[i]  ≡  *(p + i)
```

- 配列アクセスもポインター演算も、言語レベルでは同じ
- だから `arr[2]` も `p[2]` も同じ動作

### 配列 → vector のコピー

```cpp
std::array<int, 5> a = {1,2,3,4,5};

// ① 新規作成（一番シンプル）
std::vector<int> v(a.begin(), a.end());

// ② 既存 vector に上書き
v.assign(a.begin(), a.end());

// ③ std::copy（事前サイズ確保必要）
std::copy(a.begin(), a.end(), v.begin());

// ④ back_inserter（サイズ未確保でOK）
std::copy(a.begin(), a.end(), std::back_inserter(v));
```

- `std::copy` はイテレーター経由で動作、連続メモリ+trivialな型なら `memcpy` に最適化

## C#との比較

| 項目 | C# | C++ |
|---|---|---|
| ジェネリクス宣言 | `class Box<T>` | `template<typename T> struct Box` |
| 型制約 | `where T : IComparable` | コンセプト（C++20）または SFINAE |
| 型ごとのコード生成 | 実行時にジェネリクス情報が残る | コンパイル時に**型ごとに別の実体**が生成される |
| 演算子オーバーロード | `public static X operator +(...)` | `X operator+(...)`（メンバー or 非メンバー） |
| 前置/後置の区別 | なし（同じメソッド） | `operator++()` vs `operator++(int)` |
| メンバーの const 修飾 | `readonly struct` で対応 | メソッド単位で `const` 付けられる |
| イテレーター | `IEnumerator<T>`（前方固定） | 5段階のカテゴリー |
| `--` 操作 | リスト・配列ともに不可 | ランダムアクセス/双方向なら可 |
| ランダムアクセス | `IList<T>` の `[i]` | `operator[]` + イテレーター |
| 配列 | `int[]` は長さ情報付き | `int[5]` は decay でポインター化する |
| `noexcept` | `throws` なし言語（相当物なし） | パフォーマンスに影響する明示的宣言 |

## 補足

### ポインターとイテレーターの関係

```
        イテレーター（抽象：進める・参照できるもの）
               │
    ┌──────────┼──────────┐
    │          │          │
ポインター  vectorの    listの
           iterator   iterator
```

- **ポインターはイテレーターの一種**（C++標準でランダムアクセスイテレーターの要件を満たす）
- `vector`/`array` のイテレーターは**内部実装がポインター**のことも多い
- `list` のイテレーターは「ノードをたどる構造体」で、ポインターではない

### コンテナの使い分け早見表

| コンテナ | 構造 | 用途 |
|---|---|---|
| `std::array<T,N>` | 固定長配列 | サイズ固定、スタック配置 |
| `std::vector<T>` | 動的配列 | **迷ったらこれ**（実務8割） |
| `std::deque<T>` | 分割配列 | 両端追加・削除が多い |
| `std::list<T>` | 双方向リンク | 中間への挿入・削除が多い |
| `std::forward_list<T>` | 単方向リンク | メモリ節約、前方のみ |
| `std::map<K,V>` | 赤黒木 | キー検索、ソート済み |
| `std::unordered_map<K,V>` | ハッシュ表 | キー検索、順不問、高速 |
| `std::set<T>` | 赤黒木 | 重複なし集合、ソート済み |
| `std::unordered_set<T>` | ハッシュ表 | 重複なし集合、順不問 |

### イテレーターカテゴリーの覚え方

**データ構造の物理的形状**がそのまま能力制限：

| データ構造 | イテレーター |
|---|---|
| 一方通行の流れ（ファイル） | 入力 |
| 一方向リンク | 前方 |
| 双方向リンク | 双方向 |
| 連続配列 | ランダムアクセス |

カテゴリーを暗記するより **「なぜその制限があるか」** を理解する方が頭に残る。

### アルゴリズムの要求カテゴリー

| やりたいこと | 必要なカテゴリー |
|---|---|
| 検索 `std::find` | 入力 |
| コピー `std::copy` | 入力 + 出力 |
| 逆順 `std::reverse` | 双方向 |
| ソート `std::sort` | ランダムアクセス |
| 二分探索 `std::binary_search` | ランダムアクセス |

### 「一発ジャンプ」は `vector`/`array` の特権

```cpp
// vector: O(1) でジャンプ
auto it = v.begin() + 5;

// list: 物理的に不可
auto it = l.begin() + 5;   // ❌ エラー
std::advance(it, 5);       // 内部で ++ を5回（O(n)）
```

**見た目が同じでも実行速度は違う**。コンテナの特性を理解して使う。

### array-to-pointer decay の注意点

関数引数の落とし穴：

```cpp
void func(int arr[5]) {   // 配列に見えて…
    sizeof(arr);           // 8（ポインターのサイズ！）
}

// 以下と完全に同じ意味
void func(int* arr) { ... }
void func(int arr[]) { ... }
```

- 関数内では**サイズ情報が失われる**
- サイズも渡すか、`std::array` / `std::vector` を使う方が安全
