# 第27章: リファレンスとポインター

## 要点

### リファレンス（`T&`）

- 変数の**別名**を作る仕組み
- **宣言と同時に初期化必須**
- **初期化後に参照先を変更できない**（付け替え不可）
- null にできない

```cpp
int x = 10;
int& ref = x;   // x の別名

ref = 20;       // x が 20 になる（参照先の値を書き換え）
// int& r;      // ❌ 初期化なしは不可
```

### ポインター（`T*`）

- 変数の**アドレス**を保持する
- 付け替え可能、`nullptr` 可

```cpp
int x = 10;
int* p = &x;    // x のアドレスを持つ

*p = 20;        // 参照先の値を書き換え（デリファレンス）
p = &other;     // 別の変数を指すように変更
```

### `&` と `*` の2つの意味

| 文脈 | `&` | `*` |
|---|---|---|
| 宣言 | リファレンス型を表す（`int& r`） | ポインター型を表す（`int* p`） |
| 式 | アドレス取得演算子（`&x`） | デリファレンス演算子（`*p`） |

### `nullptr`

- 「どこも指していない」ことを表すポインター値（C++11以降）
- `std::nullptr_t` 型で**型安全**
- デリファレンスは**未定義動作**（例外は出ない）

```cpp
int* p = nullptr;
if (p == nullptr) { ... }
if (!p) { ... }               // bool 変換で false 扱い

*p = 10;   // ❌ 未定義動作（多くの環境でクラッシュ）
```

- `NULL`（C互換マクロ）や `0` より `nullptr` を使う

```cpp
void f(int);
void f(int*);

f(NULL);     // f(int) が呼ばれてしまう（NULLは0なので整数扱い）
f(nullptr);  // ✅ f(int*) が呼ばれる
```

### `using` による型エイリアス

- 型に別名をつける仕組み。`typedef` の改良版
- テンプレートにも使える（`typedef` では不可）

```cpp
using IntPtr = int*;
IntPtr p = nullptr;

// 長い型名を短く
using ScoreMap = std::map<std::string, std::vector<int>>;

// 関数ポインターを読みやすく
using BinaryOp = int(*)(int, int);

// テンプレートエイリアス
template <typename T>
using Vec = std::vector<T>;
Vec<int> v;
```

- **メリット**: 可読性向上、意図の明示、型変更の一元化

### スコープのルール

- `using` は**宣言した場所のスコープ**に従う
- ヘッダーに書けば `#include` した全ファイルで使える

### `const` とポインター

- 3パターンある。**右から左に読む**と理解しやすい

```cpp
int x;

const int* p1 = &x;     // 指す先が const（値を変えられない）
int const* p1b = &x;    // 同上

int* const p2 = &x;     // ポインター自体が const（付け替え不可）

const int* const p3 = &x;   // 両方 const
```

- `*` より**左**の `const` → 参照先が const
- `*` より**右**の `const` → ポインター自身が const

### ポインターのポインター

- `*` を付けるごとに一段先をたどる

```cpp
int x = 10;
int* p = &x;
int** pp = &p;

pp     // p のアドレス
*pp    // p の値（= x のアドレス）
**pp   // x の値（10）

**pp = 99;   // x が 99 になる
```

### 関数ポインター

- 関数のアドレスを変数として持ち回る
- C#の `Func`/`Action` デリゲートに相当

```cpp
int add(int a, int b) { return a + b; }

// 宣言: 戻り値型 (*変数名)(引数型リスト)
int (*op)(int, int) = add;   // & は省略可
op(1, 2);   // 3

// using で読みやすく
using BinaryOp = int(*)(int, int);
BinaryOp op = add;
```

- **用途**: コールバック、戦略の差し替え、イベント登録、関数テーブル
- モダンC++では `std::function` やテンプレート+ラムダの方が主流

### 配列へのポインター

```cpp
int arr[5] = {1,2,3,4,5};

// ① 先頭要素へのポインター（普段使うのはこっち）
int* p = arr;          // 配列名は自動で先頭ポインターに変換
p[0], p[1]

// ② 配列全体へのポインター（レア）
int (*pa)[5] = &arr;
(*pa)[0], (*pa)[1]
```

### クラスへのポインター

- そのままアクセスとの違い：**寿命・付け替え・null可否**

```cpp
User u{"Alice"};       // 実体を所有
User* p = &u;          // アドレスを指す

u.name    = "Bob";     // 直接
p->name   = "Bob";     // ポインター経由（アロー演算子）
(*p).name = "Bob";     // 同上（冗長）
```

- 関数の引数として特に重要

```cpp
void f(User u)         { ... }   // ❌ コピー発生
void f(User& u)        { ... }   // ✅ 書き換え可
void f(const User& u)  { ... }   // ✅ 読み取り専用（一番多用）
void f(User* u)        { ... }   // ✅ null を許したい時
```

### メンバーポインター（特殊用途）

- クラス内の**メンバーの位置**を指す（実体ではない）
- 使うときにオブジェクトを組み合わせる

```cpp
struct Object { int data_member; };

int Object::* mp = &Object::data_member;   // メンバーの"位置"

Object a, b;
a.*mp = 10;    // a.data_member に代入
b.*mp = 20;    // b.data_member に代入

Object* ptr = &a;
ptr->*mp = 30;   // ポインター経由
```

- 実務での使用頻度は低い。ライブラリ実装者向け

### アロー演算子 `->`

- ポインター経由でメンバーアクセスする際の略記
- `p->x` は `(*p).x` と同じ

```cpp
Point* p = &point;
p->x = 10;       // (*p).x = 10 と同じ
```

| 左辺の型 | アクセス演算子 |
|---|---|
| 値・リファレンス | `.` |
| ポインター | `->` |

## C#との比較

| 項目 | C# | C++ |
|---|---|---|
| 参照型 | すべてのクラスは参照型 | 明示的に `T&` または `T*` を使う |
| リファレンスの付け替え | `ref` パラメータ（メソッド内では不可） | **不可**（リファレンスは初期化後固定） |
| null の表現 | `null` | `nullptr` |
| null アクセス | `NullReferenceException` | **未定義動作**（例外なし） |
| リファレンスの null | null許容参照型で制御 | **リファレンスは null 不可**、ポインターのみ可能 |
| 型エイリアス | `using Age = int;`（ファイル内） | `using Age = int;`（スコープに従う） |
| 関数の値渡し | デリゲート（`Func`/`Action`） | 関数ポインター / `std::function` |
| メンバーアクセス | `.` で統一 | 値は `.`、ポインターは `->` |

## 補足

### リファレンス vs ポインターの使い分け

| 状況 | 推奨 |
|---|---|
| 関数引数で値を読むだけ（大きな構造体） | `const T&` |
| 関数引数で書き換えたい | `T&` |
| null の可能性がある | `T*` |
| 動的にオブジェクトを作る・寿命を管理 | `T*`（モダンでは `std::unique_ptr`） |
| 多態的に扱う（継承・仮想関数） | `T*` または `T&` |

### 型安全とは

- **型の不整合をコンパイラが検出してくれる性質**
- 間違った型の使い方を実行前（コンパイル時）に弾ける
- `nullptr` が `NULL` より優れているのは、ポインター専用の型で誤用を防げるから

### 宣言スタイル（`*` と `&` の位置）

```cpp
int* p;   // 型寄せ（C++コミュニティで主流）
int *p;   // 変数寄せ（C言語風）
```

**落とし穴**: カンマ区切りの複数宣言

```cpp
int* a, b;    // a はポインター、b は普通の int！
int *a, *b;   // 両方ポインターにしたいなら個別に *
```

### C#にもポインターは書けるのか？

- **書けるが、通常は使わない**
- `unsafe` ブロック内でのみ使用可能。コンパイル時に `<AllowUnsafeBlocks>true</AllowUnsafeBlocks>` 設定が必要

```csharp
unsafe {
    int x = 10;
    int* p = &x;
    *p = 20;
}
```

**主な制約**:

| 制約 | 内容 |
|---|---|
| アンマネージド型のみ | `int`, `double`, 参照型フィールドを持たない `struct` |
| クラスには直接不可 | `MyClass* p = &obj;` はコンパイルエラー |
| GC対策が必要 | クラスのフィールドを指すには `fixed` でピン留め |

```csharp
// クラスのフィールドへのポインターは fixed が必要
unsafe {
    fixed (int* p = &obj.x) {
        *p = 42;
    }
}
```

**関数ポインター**（C# 9.0以降）:

```csharp
unsafe {
    delegate*<int, int, int> op = &Add;
    int result = op(1, 2);
}
```

**C#がポインターを避ける理由**:

- GCがオブジェクトを動かすので、アドレスが変わる可能性がある
- メモリ安全性を言語設計で担保している
- `ref`、`Span<T>`、`Memory<T>` などで大抵代替できる

**使いどころ**: P/Invoke でのネイティブ相互運用、高速な数値・画像処理など。通常のアプリ開発ではほぼ登場しない。

**C#との温度差**:

| | C# | C++ |
|---|---|---|
| ポインター | 緊急避難用の裏口 | 普段使いの道具 |
| クラスへの直接ポインター | 不可 | 可 |
| 主流の代替 | `ref`, `Span<T>` | リファレンス（`T&`） |

C#でクラスを扱うとき、**実はすでに内部的に参照越しに触っている**という感覚が、C++のリファレンス/ポインターを学ぶうえで役立つ。

### キビバイト・メビバイト（2進数ベースの単位）

| 10進（SI） | 2進（IEC） | 正確な値 |
|---|---|---|
| 1 KB = 10³ | **1 KiB** = 2¹⁰ | 1,024 |
| 1 MB = 10⁶ | **1 MiB** = 2²⁰ | 1,048,576 |
| 1 GB = 10⁹ | **1 GiB** = 2³⁰ | 1,073,741,824 |

- 「1 KB = 1000? 1024?」という曖昧さ解消のためIEC が制定
- HDDメーカーは10進（1TB = 10¹²）、OSは2進で表示するため容量のズレが生じる

参考: https://en.wikipedia.org/wiki/Binary_prefix

## 理解度

- [x] リファレンスは初期化必須、付け替え不可
- [x] ポインターはアドレスを持ち、付け替え可、null 可
- [x] `nullptr` は型安全（`NULL` や `0` と違ってポインター専用）
- [x] `using` は型エイリアスで、可読性・保守性を向上させる
- [x] `const` とポインターの3パターン（右から読む）
- [ ] ポインターのポインターの使いどころ（読めればOK、書くのはレア）
- [ ] 関数ポインターの文法（`using` エイリアスで書くのが実用的）
- [ ] 配列へのポインターの文法（普段は先頭ポインターのみ使う）
- [x] クラスへのポインター：値と比べて寿命・付け替え・null可否で使い分ける
- [ ] メンバーポインター（実務ではレア、出てきたら調べる）
- [x] アロー演算子 `->` は `(*p).x` の略記
