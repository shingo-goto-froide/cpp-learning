# 第13章: 名前空間と関数

## 要点

### 関数の宣言と定義

- **定義は宣言を兼ねる**ので、別途宣言しなくてもよい

```cpp
// 定義だけでOK
int add(int a, int b) {
    return a + b;
}

int main() {
    add(1, 2); // OK
}
```

- ただしC++は**上から順に読む**ため、呼び出しより後に定義がある場合は宣言が必要

```cpp
int add(int a, int b); // 宣言（プロトタイプ）

int main() {
    add(1, 2); // OK — 宣言済み
}

int add(int a, int b) { // 定義
    return a + b;
}
```

### ヘッダーファイルによる宣言の分離

- 実務では**宣言（.h）と定義（.cpp）を分ける**のが一般的

```cpp
// math.h（宣言だけ）
int add(int a, int b);
int mul(int a, int b);

// math.cpp（定義）
int add(int a, int b) { return a + b; }
int mul(int a, int b) { return a * b; }

// main.cpp
#include "math.h"
int main() {
    add(1, 2); // OK
}
```

- 複数の`.cpp`ファイルから同じ関数を使いたいとき、ヘッダーを`#include`するだけで済む
- ヘッダーを見れば「何の関数があるか」が一覧できる

### 名前空間のネスト

- 名前空間は入れ子にできる

```cpp
namespace A {
    namespace B {
        namespace C {
            int value = 42;
        }
    }
}

A::B::C::value; // アクセス
```

- C++17からは短く書ける

```cpp
namespace A::B::C {
    int value = 42;
}
```

### 名前空間のエイリアス

- 長い名前空間に短い別名をつけられる

```cpp
namespace A::B::C {
    int value = 42;
}

namespace abc = A::B::C;
abc::value; // A::B::C::value と同じ
```

### inline名前空間

- `inline`をつけた名前空間の中身は、親の名前空間から直接アクセスできる

```cpp
namespace MyLib {
    inline namespace v2 {
        int func() { return 2; }
    }
    namespace v1 {
        int func() { return 1; }
    }
}

MyLib::func();     // 2（inlineのv2が使われる）
MyLib::v2::func(); // 2（明示的にもOK）
MyLib::v1::func(); // 1（古い方も残っている）
```

- ライブラリのバージョン管理に使う
- `inline`を付け替えるだけで、利用者のコードを変えずにデフォルトバージョンを切り替えられる
- ライブラリを作る側の機能なので、使う側は知っておく程度でOK

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 宣言と定義 | 常に一体 | 分けられる（ヘッダー + ソース） |
| ヘッダーファイル | なし | `.h` / `.hpp` |
| 他ファイルから使う | `using`で名前空間指定 | `#include`でヘッダー読み込み |
| メソッドの順番 | クラス内なら順不同 | 上から順に読むため、宣言か定義が先に必要 |
| 名前空間のネスト | `namespace A.B.C;`（C# 10） | `namespace A::B::C`（C++17） |
| 名前空間の区切り | `.` | `::` |
| 名前空間エイリアス | `using abc = A.B.C;` | `namespace abc = A::B::C;` |
| inline名前空間 | なし（NuGetバージョンで管理） | あり（バージョン切り替え用） |

## 補足

### なぜC++はヘッダーファイルが必要なのか

- C++のコンパイラは各`.cpp`ファイルを独立にコンパイルする（分割コンパイル）
- あるファイルから別ファイルの関数を呼ぶには、その関数の宣言が必要
- ヘッダーファイルは宣言を共有するための仕組み
- C#はコンパイラがプロジェクト全体を一括で処理するため、この仕組みが不要

### 学習段階での使い分け

- 1ファイルで書いている間は定義だけで十分
- ファイルを分割するようになったらヘッダーの出番
