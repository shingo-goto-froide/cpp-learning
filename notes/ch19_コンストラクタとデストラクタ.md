# 第19章: コンストラクタとデストラクタ

## 要点

### クラスの初期化

- C#と違って `new` は不要。変数宣言するだけでオブジェクトが作られる

```cpp
struct S {
    int x;
    int y;
};

S s;          // デフォルト初期化（x, yは不定値）
S s{};        // ゼロ初期化（x=0, y=0）
S s{1, 2};   // x=1, y=2
```

### デストラクタ

- スコープを抜けたら**自動で呼ばれる**。明示的に呼ぶ必要がない
- C++にはGCがないので、リソースの後片付けをデストラクタで行う

```cpp
class FileReader {
    FILE* file;
public:
    FileReader(const std::string& path) {
        file = fopen(path.c_str(), "r");  // リソース確保
    }
    ~FileReader() {
        fclose(file);  // リソース解放
    }
};

{
    FileReader reader("data.txt");  // コンストラクタが呼ばれる
    // ...使う...
}  // ← スコープを抜けると自動でデストラクタが呼ばれる
```

- この仕組みを **RAII（Resource Acquisition Is Initialization）** と呼ぶ

### 委譲コンストラクタ

- コンストラクタから別のコンストラクタを呼ぶ仕組み

```cpp
struct S {
    int x;
    int y;

    S(int x, int y) : x(x), y(y) { }   // メインのコンストラクタ
    S() : S(0, 0) { }                    // 委譲
    S(int x) : S(x, 0) { }              // 委譲
};
```

### 演算子のオーバーロード

- 演算子は関数として定義できる
- フリー関数でもメンバー関数でも書ける

```cpp
struct Vec {
    int x;
    int y;
};

// フリー関数 — 両辺を引数で受ける
Vec operator+(const Vec& l, const Vec& r) {
    return Vec{l.x + r.x, l.y + r.y};
}

Vec a{1, 2};
Vec b{3, 4};
Vec c = a + b;          // 演算子として呼ぶ
Vec c = operator+(a, b); // 関数として呼ぶ — 同じ意味
```

- 引数は `const&` が推奨（コピーしない＆変更しない）
- 戻り値の型は自由だが、直感に合う型を返すのが基本

### フリー関数とメンバー関数

```cpp
struct Vec {
    int x;
    int y;

    // メンバー関数 — クラスの中
    int length() { return x + y; }
};

// フリー関数 — クラスの外
void print(const Vec& v) {
    std::cout << v.x << ", " << v.y;
}

Vec v{1, 2};
v.length();   // メンバー関数 — v.で呼ぶ
print(v);     // フリー関数 — vを引数で渡す
```

| | メンバー関数 | フリー関数 |
|---|---|---|
| 定義場所 | クラスの中 | クラスの外 |
| 呼び方 | `v.length()` | `print(v)` |
| `this` | ある | ない |
| privateアクセス | できる | できない |
| アクセス修飾子 | ある | ない（どこからでも呼べる） |

### 前置インクリメントとlvalue

- 前置 `++i` はlvalue（参照）を返すので連続適用できる
- 後置 `i++` はrvalueを返すので連続適用できない

```cpp
int i = 0;
++++i;   // OK — ++(++i) で i == 2
i++++;   // エラー — i++はrvalue
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| オブジェクト生成 | `new S()` が必須 | `S s{};` でOK（newは不要） |
| デストラクタ | ほぼ使わない（GCがある） | スコープを抜けたら自動で呼ばれる |
| リソース解放 | `IDisposable` + `using` | デストラクタ（RAII） |
| `using` 忘れ | Disposeが呼ばれない | スコープで自動なので忘れようがない |
| 委譲コンストラクタ | `: this(...)` | `: コンストラクタ名(...)` |
| operator+ | `static` メソッド | フリー関数 or メンバー関数 |
| フリー関数 | 存在しない（必ずクラスに属す） | 普通に存在できる |
| `new` の使用 | 必須 | ほぼ使わない（スタック変数で十分） |

### newをいつ使うか（C++）

| 場面 | 書き方 |
|---|---|
| 普通に使う | `S s{1, 2};` |
| ヒープに確保したい | `std::make_unique<S>(1, 2)` |
| 生の `new` | ほぼ使わない |

## 補足

### RAIIとは

- Resource Acquisition Is Initialization（リソース確保は初期化時に）
- コンストラクタでリソースを確保し、デストラクタで解放するパターン
- C#の `using` + `IDisposable` に相当するが、C++ではスコープで自動的に管理されるのでより安全
- `new` / `delete` を手動管理するとミスが起きやすいため、RAIIやスマートポインタで管理するのが現代C++の考え方

### 演算子オーバーロードのフリー関数 vs メンバー関数

- フリー関数: 左辺と右辺を対称に扱える
- メンバー関数: 左辺が `this` になる
- どちらでも書けるが、フリー関数のほうが好まれることがある
