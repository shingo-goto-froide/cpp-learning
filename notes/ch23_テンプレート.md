# 第23章: テンプレート

## 要点

### 関数テンプレート

- `template <typename T>` を関数の前に書くと、任意の型に対応できる関数になる
- 型ごとに同じ処理を書く必要がなくなる

```cpp
// テンプレートなし → 型ごとにコピペが必要
int max_value(int a, int b) { return (a > b) ? a : b; }
double max_value(double a, double b) { return (a > b) ? a : b; }

// テンプレートあり → 1回書くだけ
template <typename T>
T max_value(T a, T b) {
    return (a > b) ? a : b;
}

max_value(3, 5);       // T = int と推論
max_value(1.2, 3.4);   // T = double と推論
max_value<int>(3, 5);  // 明示的に指定もできる
```

### `T` は慣習的な名前

- `T` は **Type** の略。名前は何でもよい
- 複数の型パラメータには `T`, `U`, `V` や `Key`, `Value` などを使う

```cpp
template <typename Hoge>
Hoge max_value(Hoge a, Hoge b) {
    return (a > b) ? a : b;
}
// これでも普通に動く
```

### C++20 の `auto` による省略

- シンプルな関数テンプレートは `auto` で短く書ける

```cpp
auto max_value(auto a, auto b) {
    return (a > b) ? a : b;
}
```

- ただし各引数が独立した型になるため、**型を揃える制約がかけられない**

```cpp
// auto — a と b が別の型でも通る
max_value(3, 2.5);   // OK: int と double

// template — 同じ型を強制できる
template <typename T>
T max_value(T a, T b) { return (a > b) ? a : b; }
max_value(3, 2.5);   // エラー！ T が int か double か曖昧
```

### クラステンプレート

- クラスにも `template` を使える。`auto` では代替できない

```cpp
template <typename T>
class Stack {
    std::vector<T> data;
public:
    void push(T value) { data.push_back(value); }
    T pop() {
        T top = data.back();
        data.pop_back();
        return top;
    }
};

Stack<int> intStack;
Stack<std::string> strStack;
```

- 標準ライブラリの多くがクラステンプレート

```cpp
std::vector<int>            // vector<T>
std::array<double, 5>       // array<T, N> ← 型以外も渡せる
std::map<std::string, int>  // map<Key, Value>
```

### `auto` と `template` の使い分け

| 場面 | 使うもの |
|---|---|
| シンプルな関数 | `auto` でOK |
| 型を揃えたい・制御したい | `template <typename T>` |
| 戻り値の型を明示したい | `template <typename T>` |
| クラス | `template` 必須 |

## C#との比較

| 項目 | C# ジェネリクス | C++ テンプレート |
|---|---|---|
| 構文 | `T MaxValue<T>(T a, T b)` | `template <typename T>` を前の行に書く |
| 解決タイミング | 実行時（JITが処理） | **コンパイル時**（型ごとにコード生成） |
| 制約 | `where T :` で明示が必要 | なくても書ける（エラーは使用時に出る） |
| コード膨張 | なし（1つのコードを共有） | 型ごとに実体化→バイナリ増加 |
| クラス | `class Stack<T>` | `template <typename T> class Stack` |

- C++のテンプレートはコンパイル時に展開されるため、`max_value<int>` と `max_value<double>` で**別々の関数が生成される**
- C#はジェネリクスの型情報を保持したまま1つのコードを共有する

## 補足

### unsigned int について

- `unsigned int` は符号なし整数。自分から積極的に使う場面は少ない
- 標準ライブラリが `std::size_t`（unsigned）を多用するため、意図せず符号付き/符号なしの混合演算が起きやすい
- 引き算でアンダーフロー（巨大な値にラップアラウンド）するバグの温床になる

```cpp
unsigned int a = 3;
unsigned int b = 5;
auto result = a - b; // 意図せず巨大な値になる
```

- Google C++ Style Guide は「unsigned は使うな」と明言
- **基本は `int` を使い、unsigned が必要な明確な理由があるときだけ使う**

## 理解度

- [x] テンプレートは「型だけ違って処理は同じ」を1回で書ける仕組み
- [x] `T` は Type の略で、名前は自由に変えられる
- [x] C++20 の `auto` で省略できるが、型の制約がかけられない
- [x] クラステンプレートは `auto` では代替できない
- [x] `std::vector` 等の標準ライブラリはクラステンプレートの恩恵
- [x] C#のジェネリクスとの違い（コンパイル時展開 vs 実行時）
