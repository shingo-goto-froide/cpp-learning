# 第38章: リテラルと文字列

## 要点

### リテラルとは

**ソースコードに直接書かれた値そのもの**。変数や式の評価結果ではなく、書いたまま意味になる値。コンパイル時に値が確定している。

```cpp
int    a = 42;         // 42 が int リテラル
double b = 3.14;       // 3.14 が浮動小数点リテラル
char   c = 'A';        // 'A' が文字リテラル
bool   d = true;       // true が真偽値リテラル
auto   s = "hello";    // "hello" が文字列リテラル
```

### リテラルの種類

| 種類 | 例 | 型 |
|---|---|---|
| 整数リテラル | `42`, `0xFF`, `0b1010`, `100'000` | `int` 等 |
| 浮動小数点リテラル | `3.14`, `1.5e10`, `2.0f` | `double`, `float` |
| 文字リテラル | `'A'`, `'\n'`, `'\''` | `char` |
| 文字列リテラル | `"hello"`, `"a\nb"` | `const char[N]` |
| 真偽値リテラル | `true`, `false` | `bool` |
| ポインタリテラル | `nullptr` | `std::nullptr_t` |
| ユーザー定義リテラル | `42ms`, `"foo"sv` | 任意 |

### 文字リテラルと文字列リテラルは別物

| 項目 | 文字リテラル | 文字列リテラル |
|---|---|---|
| 囲み | シングル `'...'` | ダブル `"..."` |
| 内容 | **1文字だけ** | 0文字以上 |
| 型 | `char` | `const char[N]` |
| 末尾 | なし | **`\0` が自動付加** |
| 例 | `'A'` | `"A"` |

```cpp
char a = 'A';            // 1文字、サイズ 1
const char* b = "A";     // 'A' と '\0' の2バイト配列
sizeof("hello");         // 6（'\0' 含む）
sizeof('A');             // 1
strlen("hello");         // 5（'\0' は数えない）
```

### エスケープシーケンス

囲み文字自身や特殊文字をリテラル中に書きたいときに `\` を使う。

```cpp
char c = '\'';   // シングルクォート1文字
char d = '\\';   // バックスラッシュ1文字
const char* s = "He said \"hi\".";   // 文字列内のダブルクォート
```

| 書き方 | 意味 |
|---|---|
| `'\''` | シングルクォート `'` |
| `'\"'` | ダブルクォート `"` |
| `'\\'` | バックスラッシュ `\` |
| `'\n'` | 改行 (LF) |
| `'\r'` | 復帰 (CR) |
| `'\t'` | タブ |
| `'\0'` | ヌル文字 |
| `'\x41'` | 16進指定（`A` = 0x41） |

ルール：**「囲み文字と同じ記号を中に書きたいときにエスケープが必要」**。

```cpp
char c1 = '"';        // ✅ char では " はエスケープ不要
const char* s = "It's fine.";   // ✅ 文字列では ' はエスケープ不要
```

### 文字列リテラルの正体は配列

`"hello"` は **6文字の `const char` 配列**（末尾の `\0` を含む）。これを **C スタイル文字列** と呼ぶ。

```cpp
const char* s = "hello";
// メモリ：[ 'h', 'e', 'l', 'l', 'o', '\0' ]
// 型は const char[6]
```

関数に渡したり変数に入れたりするときに **配列 → 先頭ポインタへ自動変換**（配列の decay）される。

### `const` が付く理由

文字列リテラルは **読み取り専用領域** に置かれる。書き換えようとすると未定義動作。

```cpp
char* s = "hello";   // ❌ C++では非推奨／エラー（C++11以降）
s[0] = 'H';          // 💥 未定義動作
```

### `std::basic_string` への変換が必要な理由

`std::string` の正体：

```cpp
namespace std {
    using string    = basic_string<char>;
    using wstring   = basic_string<wchar_t>;
    using u8string  = basic_string<char8_t>;
    using u16string = basic_string<char16_t>;
    using u32string = basic_string<char32_t>;
}
```

`std::string` は **`basic_string<char>` のエイリアス**。文字型をテンプレート引数にした汎用クラス。

C スタイル文字列のままだと不便：

```cpp
const char* s = "hello";
s.size();           // ❌ ポインタにメソッドはない
s + " world";       // ❌ ポインタの加算（先頭から N 文字進む）
s == "hello";       // ❌ ポインタ同士の比較（中身比較ではない）
```

`std::string` にすると：

```cpp
std::string s = "hello";
s.size();              // ✅ 5（O(1)）
s + " world";          // ✅ 文字列連結
s == "hello";          // ✅ 中身の比較
s[0] = 'H';            // ✅ 書き換え可能
s.append(" world");    // ✅ 末尾追加
s.substr(1, 3);        // ✅ 部分文字列
s.find("ll");          // ✅ 検索
```

### 変換する主な理由

1. **動的に変化させたい**：リテラルは読み取り専用 → コピーして書き換える
2. **連結したい**：`+` 演算子が使える
3. **長さを高速に取りたい**：`strlen` は O(N)、`size()` は O(1)
4. **安全に管理したい**：自動拡張でバッファオーバーランを防げる
5. **コンテナと組み合わせる**：`std::vector<std::string>` などが自然
6. **STL アルゴリズムが使える**：`std::sort`, `std::reverse` 等

### 変換のコスト

`std::string` への変換は **ヒープ確保 + コピー** が走る。短い文字列なら **SSO**（Small String Optimization）でスタックに収まり高速。

### `std::string_view`（C++17）— 軽量な選択肢

書き換えないなら `std::string_view` のほうが軽い。**ポインタ + 長さ** を持つだけの参照型。所有権はない。

```cpp
void print(std::string_view s) {     // const char* / std::string 両方受ける
    std::cout << s;
}

print("hello");              // ✅ コピーなし
print(std::string("hi"));    // ✅ コピーなし
```

### 文字列リテラルの自動連結

ソースコード上で隣り合う文字列リテラルは **コンパイラが自動連結** する。

```cpp
const char* s = "hello "
                "world";
// ↑ 自動的に "hello world" として扱われる
```

### リテラルのプレフィックス・サフィックス

リテラルに修飾子を付けて型を変えられる。

```cpp
auto a = 42L;       // long
auto b = 42U;       // unsigned int
auto c = 42LL;      // long long
auto d = 3.14f;     // float
auto e = 3.14L;     // long double

auto g = "hello";   // const char*
auto h = L"hello";  // const wchar_t*
auto i = u8"hello"; // const char8_t*（UTF-8、C++20）
auto j = u"hello";  // const char16_t*
auto k = U"hello";  // const char32_t*
```

C++14 以降の `std::string` リテラル：

```cpp
using namespace std::literals;
auto s = "hello"s;        // std::string
auto sv = "hello"sv;      // std::string_view
auto t = 100ms;           // std::chrono::milliseconds
```

### raw 文字列リテラル（C++11）

エスケープを一切しない記法。

```cpp
const char* path = R"(C:\Users\foo\bar.txt)";
const char* re = R"(\d+\.\d+)";
const char* json = R"({"name": "hi"})";
```

C# の `@"..."` に相当。**ただし `char` リテラルには raw 記法はない** ので `'\''` は引き続き必要。

### 文字リテラルの中身は整数

`char` リテラルは内部的に **整数値**（文字コード）。

```cpp
char c = 'A';        // 'A' の値は 65（ASCII）
int  n = 'A';        // ✅ int として 65 が入る
char d = 65;         // ✅ 'A' と同じ
'A' + 1;             // 結果は 66（'B' に相当）
```

## C#との比較

| 項目 | C# | C++ |
|---|---|---|
| 文字リテラル | `'A'`（型は `char`） | `'A'`（型は `char`） |
| 文字列リテラル | `"hello"`（型は `string`） | `"hello"`（型は `const char[N]`） |
| 文字列の実体 | `string` クラス（参照型） | C スタイル配列 |
| 標準文字列型 | `string`（言語組み込み） | `std::string`（ライブラリ） |
| 不変／可変 | **不変**（immutable） | **可変**（mutable） |
| 文字コード | UTF-16（`char` は 2バイト） | 実装依存（通常 1バイト） |
| エスケープ無効化 | `@"..."` | `R"(...)"` |
| 文字列補間 | `$"hello {name}"` | なし（C++20 の `std::format` で代替） |
| 連結 | `+` 演算子 | リテラル隣接で自動／`std::string` で `+` |
| 軽量参照 | `ReadOnlySpan<char>` | `std::string_view` |
| 長さ取得 | `.Length`（プロパティ） | `.size()`（メソッド） |
| リテラルにメソッド呼び出し | `"hello".Length` ✅ | リテラル単体には不可 |

C# は「**文字列はオブジェクト**」が前提。`"hello"` と書いた瞬間に `string` 型なので、変換という概念がない。
C++ は「**文字列は char の配列**」が基本。`std::string` を使うには明示的に変換する。

```csharp
// C#：リテラルから直接機能が使える
"hello".Length;
"hello" + " world";
```

```cpp
// C++：std::string にして初めて全機能が使える
std::string s = "hello";
s.size();
s + " world";
```

### C# の文字列の不変性との違い

C# の `string` は **immutable**。`s + " world"` は新しい `string` を作って返す。
C++ の `std::string` は **mutable**。書き換え可能。

```csharp
string s = "hello";
s += " world";   // s は新しい string オブジェクトに付け替えられる（元は破棄）
```

```cpp
std::string s = "hello";
s += " world";   // s 自身が拡張される（同じオブジェクト）
```

## 補足

### `std::string` がテンプレートクラスである理由

文字型ごとに別クラスを作るのは無駄。1つのテンプレートで多種対応できる。

```cpp
std::basic_string<char>      // = std::string（1バイト文字）
std::basic_string<wchar_t>   // = std::wstring（Windows API 等で使用）
std::basic_string<char8_t>   // = std::u8string（UTF-8、C++20）
```

国際化や OS 依存の API（Windows は `wchar_t`）を扱うときに切り替えられる。

### 関係性の全体像

```
                  ソースコード
                       │
              ┌────────┴────────┐
              │                 │
        単一文字            複数文字（文字列）
        'A', '\n'           "hello", "a\nb"
              │                 │
        char リテラル      文字列リテラル
        型: char           型: const char[N]
              │                 │
              ↓                 ↓
        char 変数           ・const char* で受ける（C スタイル）
                            ・std::string にコピーして使う（C++ 流）
                            ・std::string_view で参照（C++17）
```

### 変換するかどうかの指針

| 状況 | 選択 |
|---|---|
| 関数引数で読むだけ | `std::string_view`（C++17 以降）または `const char*` |
| 関数引数で書き換える／所有する | `std::string`（値渡し or `&`） |
| ローカルで操作する | `std::string` |
| メンバとして保持する | `std::string` |
| グローバル定数 | `const char*` または `constexpr std::string_view` |

### `std::string` のメモリ管理

短い文字列はスタック上に収まる（SSO）、長くなるとヒープに展開。

```cpp
std::string s = "hi";              // SSO で済む（ヒープ確保なし）
std::string t = "this is a much longer string ...";  // ヒープに確保
```

実装依存だが、多くの実装では 15〜22 文字程度までは SSO で動く。

### `'\0'` の役割と落とし穴

C スタイル文字列は **`\0` を見つけたら終わり** とみなす。途中に `\0` が混じると、それ以降が無視される。

```cpp
const char* s = "hello\0world";
strlen(s);           // 5（最初の \0 で終了）
std::cout << s;      // "hello" まで出力
```

`std::string` は内部に長さを保持しているので `\0` 入りでも扱える。

```cpp
std::string t("hello\0world", 11);   // 長さを明示
t.size();   // 11
```

### 数値リテラルの区切り（C++14）

長い数値リテラルは `'`（シングルクォート）で区切れる。

```cpp
auto million = 1'000'000;
auto hex = 0xFF'FF'FF'FF;
auto bin = 0b1010'1100;
```

可読性向上のための機能。値は変わらない。

### 文字列リテラル比較の罠

```cpp
const char* a = "hello";
const char* b = "hello";
a == b;            // △ 実装依存（同じリテラルプールに置かれると true）

std::string s = "hello";
s == "hello";      // ✅ 中身比較（true）
strcmp(a, b) == 0; // ✅ 中身比較
```

`const char*` 同士の `==` はポインタ比較。中身比較は `std::string` か `strcmp` を使う。

### `std::string` のコンストラクタは多彩

```cpp
std::string a;                       // 空
std::string b("hello");              // C 文字列から
std::string c(5, 'x');               // "xxxxx"
std::string d(b, 1, 3);              // b の [1,3) → "ell"
std::string e{'h','i'};              // 初期化リスト
std::string f(b.begin(), b.end());   // イテレータから
```

### ユーザー定義リテラル（参考）

C++11 以降、自分でリテラルのサフィックスを定義できる。

```cpp
constexpr long double operator""_kg(long double x) { return x; }
constexpr long double operator""_g(long double x) { return x / 1000.0; }

auto w = 5.0_kg + 200.0_g;   // 5.2（kg）
```

## 理解度

- [x] リテラルの定義（ソースコードに直接書く値）
- [x] リテラルの種類（整数／浮動小数点／文字／文字列／真偽値／ポインタ）
- [x] 文字リテラル `'A'` と文字列リテラル `"A"` の違い
- [x] 文字列リテラルの実体は `const char[N]`（末尾に `\0`）
- [x] エスケープシーケンス（`'\''`、`'\\'`、`'\n'` など）
- [x] エスケープが必要な場面（囲み文字と中身が衝突するとき）
- [x] `std::string` は `std::basic_string<char>` のエイリアス
- [x] リテラルのままだとメソッドや演算子が使えない
- [x] `std::string` への変換で得られる機能
- [x] 変換のコスト（ヒープ確保＋コピー）と SSO
- [x] `std::string_view` は読み取り専用の軽量参照
- [x] 文字列リテラルの自動連結
- [x] プレフィックス（`L`, `u8`, `u`, `U`, `R`）と サフィックス（`s`, `sv`, `f`, `L`）
- [x] raw 文字列リテラル `R"(...)"` の使い方
- [x] 文字リテラルは整数値として扱える
- [x] C# の `string`（不変・組み込み）との違い
