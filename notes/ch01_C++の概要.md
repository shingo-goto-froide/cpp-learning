# 第1章: C++の概要

## 要点

### コンパイル方式の違い

| 方式 | 説明 | 言語例 |
|------|------|--------|
| **AOTコンパイル** | 実行前にネイティブコードへ変換。実行速度が速い | C, C++ |
| **JITコンパイル** | 実行時に中間コードからネイティブコードへ変換 | C#, Java |

- C++ は AOT（Ahead-Of-Time）コンパイル → 直接機械語になるため高速
- C# / Java は JIT（Just-In-Time）コンパイル → 中間言語（IL / バイトコード）を経由

> **C#との比較:**
> C# では `.cs` → IL（中間言語）→ CLR が JIT で実行、という流れ。
> C++ では `.cpp` → 直接ネイティブコード（`.exe`）。間に VM やランタイムが挟まらない。
> そのため C++ は配布時に .NET ランタイムが不要だが、プラットフォームごとに再コンパイルが必要。

### メモリ管理

- C++ には **GC（ガベージコレクション）がない**
- 代わりに以下の手段でメモリを管理する:
  - **手動管理**（`new` / `delete`）
  - **スマートポインタ**（`std::unique_ptr`, `std::shared_ptr` 等）
  - **RAII**（Resource Acquisition Is Initialization）

> **C#との比較:**
> C# では GC が不要オブジェクトを自動回収してくれるため、メモリ解放を意識する必要がほぼない。
> C++ では自分で管理する必要がある。ただしモダン C++ ではスマートポインタと RAII を使えば、
> C# の `using` 文（`IDisposable`）に近い感覚でリソース管理ができる。
>
> | C# | C++ |
> |----|-----|
> | `using var fs = new FileStream(...)` | `std::ifstream fs("...")` ← スコープ終了で自動 close |
> | GC が参照カウント不要オブジェクトを回収 | `std::shared_ptr` が参照カウント0で即座に解放 |

### スタックとヒープ

| 領域 | 特徴 | 見分け方 |
|------|------|----------|
| **スタック** | 関数内のローカル変数。自動で確保・解放 | 普通に宣言した変数 |
| **ヒープ** | `new` で確保。明示的に解放が必要 | `new` / スマートポインタ経由 |

> **C#との比較:**
> C# では値型（`int`, `struct`）はスタック、参照型（`class`）はヒープという区分がある。
> C++ では**すべての型がデフォルトでスタック**に置かれる。ヒープに置くには明示的に `new` する。
> つまり C# の `var obj = new MyClass()` は常にヒープだが、C++ の `MyClass obj;` はスタック。

## 補足

- C++ は「ゼロオーバーヘッド原則」を重視する言語。使わない機能にはコストがかからない設計
- GC がない代わりに RAII パターンが非常に重要。スコープを抜けると自動でリソース解放される
