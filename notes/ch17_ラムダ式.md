# 第17章: ラムダ式

## 要点

### ラムダ式の基本

- ラムダ式は**関数を定義しているだけ**で、呼び出しは別
- 引数がないラムダは `()` を省略できる

```cpp
// 引数あり
auto f = [](auto x) { std::cout << x; };
f("hello"s);

// 引数なし — ()は省略可能
auto g = [] { std::cout << "hello"; };
auto g = []() { std::cout << "hello"; };  // 同じ意味
```

### ラムダをその場で呼び出す

```cpp
[](auto x) { std::cout << x; }("hello"s);
//                              ^^^^^^^^^ ここで呼び出し
```

- 末尾の `()` で即座に呼び出している
- 実用的にはほぼ使わない。「ラムダは関数オブジェクトである」ことを示すための書き方

### キャプチャ

- ラムダの外にある変数を使うための仕組み
- `[]` の中で「どの変数を、どう取り込むか」を指定する

#### コピーキャプチャ

```cpp
int x = 10;
int y = 20;

auto f1 = [=]() { return x + y; };   // 全部コピー
auto f2 = [x]() { return x; };       // xだけコピー
auto f3 = [x, y]() { return x + y; }; // 個別に指定して両方コピー
```

#### リファレンスキャプチャ

```cpp
int x = 10;
int y = 20;

auto f1 = [&]() { x += 1; y += 1; };      // 全部参照
auto f2 = [&x]() { x += 1; };              // xだけ参照
auto f3 = [&x, &y]() { x += 1; y += 1; }; // 個別に指定して両方参照
```

#### 混在

```cpp
auto f = [&x, y]() { x += 1; };  // xは参照、yはコピー
auto f = [=, &x]() { };          // 基本コピー、xだけ参照
```

#### キャプチャ一覧

| 書き方 | 意味 |
|---|---|
| `[]` | 何もキャプチャしない |
| `[=]` | 全部コピー |
| `[&]` | 全部参照 |
| `[x]` | xだけコピー |
| `[&x]` | xだけ参照 |
| `[x, y]` | x, yをコピー |
| `[&x, &y]` | x, yを参照 |
| `[=, &x]` | 基本コピー、xだけ参照 |
| `[&, x]` | 基本参照、xだけコピー |

- `[]` で外の変数を使おうとするとコンパイルエラー
- `[=]` は楽だが意図しない変数まで取り込む可能性がある → 個別指定のほうが安全

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| ラムダ構文 | `x => x * 2` | `[](auto x) { return x * 2; }` |
| キャプチャ | 自動（全部参照） | `[]` で明示的に指定 |
| コピー/参照の選択 | 選べない（常に参照） | `[=]` か `[&]` か選べる |
| 型指定 | `Func<T>` / `Action<T>` が必要 | `auto` で済む |
| 引数なしの省略 | `() => ...` の `()` は必須 | `[]{ ... }` で `()` 省略可能 |

- C#ではキャプチャを意識する必要がなかったが、C++ではコピーか参照かを自分で決める
- これはC++の「コピーコストの管理はプログラマの責任」という設計思想に基づく

## 補足

### なぜC++はキャプチャを明示させるのか

- コピーか参照かでパフォーマンスや挙動が変わる
- 暗黙にキャプチャすると意図しない変数まで取り込んでしまう可能性がある
- `[]` を見るだけで「外の変数は使っていない」とわかるので可読性にも貢献する

### ラムダは定義であり呼び出しではない

- ラムダ式の `[](auto x) { ... }` は定義しているだけ
- 呼び出しは変数に入れて `f(...)` するか、アルゴリズムに渡して内部で呼ばれるのが一般的

```cpp
// パターン①: 変数に入れて呼び出す
auto f = [](auto x) { std::cout << x; };
f("hello"s);

// パターン②: アルゴリズムに渡す
std::for_each(std::begin(v), std::end(v), [](auto x) { std::cout << x; });
```

## 理解度

- [x] ラムダ式の基本構文（`[](){}` と `()` 省略）
- [x] ラムダは定義であり呼び出しは別
- [x] キャプチャの仕組み（外の変数を使うための指定）
- [x] コピーキャプチャ（`[=]` `[x]`）
- [x] リファレンスキャプチャ（`[&]` `[&x]`）
- [x] 混在キャプチャ（`[=, &x]` `[&x, y]`）
- [x] キャプチャなし `[]` はエラーになる理由
