# 第25章: イテレーターの実装

## 要点

### イテレーターの正体

- イテレーターは**配列への参照 + 現在位置**を持つ構造体
- `*iter`、`++iter`、`!=` はすべて演算子オーバーロード

```cpp
template <typename Array>
struct array_iterator {
    Array & a;        // どの配列を指しているか
    std::size_t i;    // 現在位置

    array_iterator(Array & a, std::size_t i)
        : a(a), i(i) {}

    // *iter — 現在位置の要素を返す
    typename Array::reference operator *() {
        return a[i];
    }

    // ++iter — 次に進む
    array_iterator & operator ++() {
        ++i;
        return *this;
    }

    // iter != other — 位置が違うか比較
    bool operator !=(array_iterator const & other) const {
        return i != other.i;
    }
};
```

### typename が必要な場面

- テンプレートパラメータの中の型を `::` で取り出すとき、`typename` が必要

```cpp
// エラー — 型か値かコンパイラが判断できない
Array::reference operator *() { return a[i]; }

// OK — 「これは型です」と明示
typename Array::reference operator *() { return a[i]; }
```

### *this — 自分自身への参照

- `*this` は「このオブジェクト自身」を指す
- イテレーターに配列自身を渡すときに使う

```cpp
struct array {
    iterator begin() {
        return iterator(*this);  // 自分自身の参照をイテレーターに渡す
    }
};
```

- コピーではなく参照を渡しているだけ（イテレーターは `Array &` で受け取る）

### begin() と end()

- `begin()` は先頭を指すイテレーター、`end()` は末尾の**1つ先**を指すイテレーターを返す

```cpp
struct array {
    iterator begin() {
        return iterator(*this, 0);          // 先頭
    }
    iterator end() {
        return iterator(*this, N);          // 末尾の1つ先
    }
};
```

```
[0] [1] [2] [3] [4] | end
begin →→→→→→→→→→→ != end で終了
```

### 範囲for の仕組み

- 範囲forは内部的に `begin()` と `end()` を呼んでいる

```cpp
// この書き方は
for (auto & x : a) {
    std::cout << x;
}

// 内部的にはこうなっている
for (auto iter = a.begin(); iter != a.end(); ++iter) {
    std::cout << *iter;
}
```

### const_iterator と iterator は別の型

- `const` な配列には `const_iterator` が必要

```cpp
template <typename Array>
struct array_const_iterator {
    Array const & a;   // const 参照で持つ
    std::size_t i;

    typename Array::const_reference operator *() const {
        return a[i];
    }
};
```

- `array` 側にも `const` 版の `begin()` / `end()` を用意する

```cpp
struct array {
    // 通常版
    iterator begin() {
        return iterator(*this);
    }

    // const 版
    const_iterator begin() const {
        return const_iterator(*this);
    }
};
```

| | `iterator` | `const_iterator` |
|---|---|---|
| 配列への参照 | `Array &` | `Array const &` |
| `*iter` | 読み書きできる | **読み取り専用** |

### 後置 ++ はコピーを返す

- 前置 `++iter` は参照を返す（変更後の自分自身）
- 後置 `iter++` はコピーを返す（変更前の値）
- C#でも同じ動作

```cpp
int a = 1;
int b = a++;  // b = 1（コピー）, a = 2
int c = ++a;  // c = 3（参照）, a = 3
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| イテレーターの仕組み | `IEnumerator<T>` インターフェース | 演算子オーバーロードを持つ構造体 |
| `*iter` 相当 | `.Current` プロパティ | `operator *()` |
| `++iter` 相当 | `.MoveNext()` メソッド | `operator ++()` |
| const の区別 | なし（`Current` は常に読み取り専用） | `iterator` と `const_iterator` の2つ |
| 範囲ループ | `foreach` — `IEnumerable<T>` を使う | 範囲for — `begin()` / `end()` を使う |
| `this` の渡し方 | 参照型なら自動的に参照 | `*this` で明示的に渡す |
| `typename` の指定 | 不要（コンパイラが自動判断） | テンプレート内で `::` で型を取り出すとき必要 |

## 補足

### 実務でのループの書き方

- **範囲for（`for (auto & x : a)`）が基本**
- インデックスが必要なときだけインデックスループ
- `begin()` / `end()` を直接書くのは `std::sort` 等のアルゴリズム関数に渡すときくらい

```cpp
// 基本はこれ
for (auto & x : a) { std::cout << x; }

// インデックスが必要なとき
for (std::size_t i = 0; i < a.size(); ++i) { std::cout << a[i]; }

// アルゴリズム関数に渡すとき
std::sort(a.begin(), a.end());
```

### この章の位置づけ

- `std::array` のイテレーターを自作することで、ライブラリの裏側を理解する章
- 実務でイテレーターを自作することはまずない
- 仕組みを知っておくとエラーが出たときに原因を推測しやすくなる

## 理解度

- [x] イテレーターは配列への参照 + 位置を持つ構造体
- [x] `*iter`、`++iter`、`!=` は演算子オーバーロード
- [x] `begin()` / `end()` はイテレーターを返すだけの関数
- [x] 範囲for は内部的に `begin()` と `end()` を使っている
- [x] `typename` はテンプレート内で `::` を使って型を取り出すとき必要
- [x] `*this` は自分自身への参照（コピーではない）
- [x] `const_iterator` と `iterator` は別の型で2セット必要
- [x] 後置 `++` はコピーを返す
