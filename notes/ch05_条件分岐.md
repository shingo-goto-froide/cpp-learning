# 第5章: 条件分岐

## 要点

### if / else if / else

- 基本構文はC#とほぼ同じ
- **C++では整数やポインタが条件式に使える**（暗黙のbool変換）

```cpp
int x = 42;
if (x) { /* 0以外はtrue、0はfalse */ }

const char* p = nullptr;
if (p) { /* nullptrはfalse */ }
```

- `std::string`はboolに暗黙変換できないため、条件式には使えない

```cpp
std::string s = "hello";
// if (s) {}       // コンパイルエラー
if (s.empty()) {}  // OK: 空文字チェックはこう書く
```

### if文での変数宣言（C++17）

```cpp
if (auto result = compute(); result > 0) {
    // resultのスコープはこのif文の中だけ
}
```

### switchのフォールスルー

- C++ではbreakを書き忘れると次のcaseに**落ちる**（C#ではコンパイルエラー）

```cpp
switch (x) {
    case 1:
        std::cout << "one";
        // break忘れ → case 2に落ちる！
    case 2:
        std::cout << "two";
        break;
}
```

### 文字列の比較

- `std::string`同士は `==`, `<`, `>` などの演算子で直接比較できる（辞書順）
- `""s`サフィックスには `using namespace std::string_literals;` が必要

```cpp
if ("cat"s > "dog"s) { /* ... */ }  // 辞書順で比較
```

- サフィックスなしの文字列リテラル同士の比較は**ポインタ比較**になるため注意

```cpp
if ("cat" > "dog") { /* ポインタ比較！結果は不定 */ }
```

### std::boolalpha

- bool値を `1`/`0` ではなく `true`/`false` で出力するマニピュレータ

```cpp
std::cout << std::boolalpha << true;   // 出力: true
std::cout << std::noboolalpha << true; // 出力: 1
```

### boolの算術演算

- C++ではboolが整数に暗黙変換されるため、算術演算が可能

```cpp
std::cout << true + true;   // 2
int count = (a > 0) + (b > 0) + (c > 0);  // 条件の数を数える
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 条件式に整数 | コンパイルエラー | OK（0=false、非0=true） |
| switchフォールスルー | コンパイルエラー | 警告なしで通ることがある |
| 文字列の大小比較 | `string.Compare()` が必要 | `<`, `>` で直接書ける |
| bool + bool | コンパイルエラー | OK（整数に暗黙変換） |
| bool出力 | `true.ToString()` → `"True"` | デフォルト `1`、`std::boolalpha`で `true` |

## 補足

- 条件式での暗黙変換はバグの原因になりやすい。意図的に使う場合以外は明示的にboolで書くのが安全
- `-Wall`オプションがあればswitchのフォールスルーに警告が出る
