# 第18章: クラス

## 要点

### classとstructの違い

- C++では**デフォルトのアクセス修飾子だけ**が違う。それ以外は完全に同じ

```cpp
class Foo {
    int x;  // デフォルトprivate
};

struct Bar {
    int x;  // デフォルトpublic
};
```

| | class | struct |
|---|---|---|
| デフォルト | private | public |
| 継承 | できる | できる |
| メソッド | 持てる | 持てる |
| メモリ | 同じ | 同じ |

### データメンバーは変数ではない

- クラス定義の時点ではメモリに存在しない（設計図のようなもの）
- オブジェクトを作って初めて実体ができる

```cpp
struct S {
    int x;  // データメンバー — この時点では実体なし
};

S a;    // ここで初めて a.x がメモリ上に作られる
S b;    // b.x は a.x とは別の実体
```

### アクセス修飾子

- C++ではクラス自体にアクセス修飾子はつけない（定義すればどこからでも使える）
- アクセス制御があるのは**メンバー（中身）だけ**

```cpp
class User {
private:
    std::string password;  // 外から触れない
public:
    std::string name;      // 外から触れる
    void login();
};
```

### データメンバーの命名慣習

- データメンバーと引数を区別するためにアンダースコアをつける

```cpp
class User {
private:
    std::string name_;   // データメンバーに末尾 _ をつける

public:
    void set_name(std::string name) {
        name_ = name;    // どっちがどっちか一目でわかる
    }
};
```

| スタイル | データメンバー | 引数 |
|---|---|---|
| 末尾アンダースコア | `name_` | `name` |
| `m_` プレフィックス | `m_name` | `name` |
| `this->` で区別 | `this->name` | `name` |

- C++では先頭アンダースコアは予約名と衝突する可能性があるため**末尾が安全**

### 継承

- C#と同じく継承ができる
- `: public Base` のように `public` を明示する

```cpp
class Animal {
public:
    void speak() { std::cout << "..."; }
};

class Dog : public Animal {
public:
    void speak() { std::cout << "wan"; }
};
```

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| class vs struct | 参照型 vs 値型（根本的に別物） | デフォルトのアクセス修飾子が違うだけ |
| structにメソッド | 書ける（が少ない） | 普通に書く |
| クラスのアクセス修飾子 | `public class` / `internal class` | なし（常にどこからでも使える） |
| メンバーの命名慣習 | `_name`（先頭アンダースコア） | `name_`（末尾アンダースコア） |
| 継承の書き方 | `class Dog : Animal` | `class Dog : public Animal` |

- C#では class = 参照型、struct = 値型 という明確な区別があるが、C++では**ほぼ同義**
- C#の `internal` / `public` のようなクラス単位の可視性制御はC++にはない

## 補足

### 入門書でstructが多い理由

- `struct` はデフォルトが public なのでアクセス修飾子を書かなくて済む
- アクセス制御を気にせず学べるように入門書では `struct` を使うことが多い
- 実務では `class` + `public` / `private` の使い分けが一般的

## 理解度

- [x] classとstructの違い（デフォルトのアクセス修飾子だけ）
- [x] データメンバーは変数ではない（オブジェクト生成で実体化）
- [x] アクセス修飾子はメンバーにつける（クラス自体にはつけない）
- [x] データメンバーの命名慣習（末尾アンダースコア）
- [x] 継承の基本（`: public Base`）
