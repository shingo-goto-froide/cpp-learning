# 第22章: 配列

## 要点

### 組み込み配列（Cスタイル配列）

- 言語に組み込まれた固定長配列
- サイズ情報を持たず、関数に渡すとポインタに化ける

```cpp
int a[5] = {1, 2, 3, 4, 5};
```

### std::array

- 標準ライブラリのクラス（組み込み配列のラッパー）
- サイズ取得・範囲外チェックができる。パフォーマンスは組み込み配列と同じ

```cpp
#include <array>
std::array<int, 5> a = {1, 2, 3, 4, 5};

a.size();   // 5
a.at(10);   // 範囲外 → 例外が飛ぶ
a[10];      // 範囲外 → チェックなし（未定義動作）
```

| 項目 | `int a[5]` | `std::array<int,5>` |
|---|---|---|
| サイズ取得 | 面倒 | `.size()` |
| 範囲外チェック | なし | `.at()` で可能 |
| 関数に渡す | ポインタに化ける | そのまま渡せる |
| イテレーター | 使える | 使える |
| パフォーマンス | 同じ | 同じ |

- **現代C++では `std::array` を使うのが推奨**

### 演算子オーバーロードで参照を返す理由

- `operator[]` が値（コピー）を返すと、代入ができない

```cpp
// 値を返す場合
int operator [](std::size_t i) {
    return data[i];
}
a[2] = 99;  // エラー！コピーに代入しようとしている

// 参照を返す場合
int & operator [](std::size_t i) {
    return data[i];
}
a[2] = 99;  // OK！data[2] そのものを書き換える
```

### 演算子ごとの参照を返す・返さないの使い分け

- **「その結果に対してさらに操作したいか？」** で決まる

| 演算子 | `&` で返す？ | 理由 |
|---|---|---|
| `[]` | ほぼ常に | 代入したいから |
| `+` `-` `*` `/` | 返さない | 新しい値を作るから |
| `=` `+=` `-=` | 返す | `a = b = c` のような連鎖代入のため |
| `++`（前置） | 返す | `++++i` のような連続適用のため |
| `++`（後置） | 返さない | 変更前の値のコピーを返すから |

## C#との比較

| 項目 | C# | C++ |
|------|-----|------|
| 配列 | `int[]` — サイズ取得・範囲外チェックあり | `int a[5]` — どちらもなし |
| 安全な配列 | `int[]` が最初から安全 | `std::array` を使う |
| `[]` の定義 | `this[int i]`（get/set別々） | `operator[]` で参照を返す |
| 参照返し | `ref return`（C# 7.0+） | `int &` を返すだけ |

- C#の配列は最初から `.Length` も範囲外チェックもあるので、`std::array` に近い

## 補足

### std::array vs std::vector

| | `std::array` | `std::vector` |
|---|---|---|
| サイズ | コンパイル時に固定 | 実行時に可変 |
| メモリ | スタック | ヒープ |
| 用途 | サイズが決まっている場合 | サイズが変わる場合 |

- サイズが変わるなら `std::vector`、固定なら `std::array` を使い分ける

## 理解度

- [x] 組み込み配列と `std::array` の違い
- [x] `std::array` が推奨される理由
- [x] `operator[]` で参照を返す理由（代入を可能にするため）
- [x] 演算子ごとの参照を返す・返さないの判断基準
