# 第3章: 変数と初期化

## 要点

### 変数名のルール

- 先頭はアルファベットまたは `_`（アンダースコア）
- 2文字目以降はアルファベット、数字、`_`
- **`_` で始まる名前は避ける** → 処理系やライブラリ内部で予約されている可能性がある
  - 特に `_` + 大文字（例: `_Value`）や `__` で始まる名前は**規格上予約済み**

### 初期化の3種類

```cpp
int a = 0;    // コピー初期化
int b(0);     // 直接初期化
int c{0};     // リスト初期化（波括弧初期化） ← 推奨
```

| 方式 | 縮小変換 | 備考 |
|------|----------|------|
| `= 値` | 許可される | 暗黙の型変換で精度が失われる可能性 |
| `(値)` | 許可される | 同上 |
| `{値}` | **エラーになる** | 安全。意図しないデータ損失を防ぐ |

**縮小変換の例:**
```cpp
int x = 3.14;   // OK（小数部分が切り捨てられる）
int y{3.14};    // エラー！ 縮小変換は許可されない
```

→ `{}` を使えば「うっかり精度を失う」ミスをコンパイル時に検出できる

> **C#との比較:**
> C# は `int x = 3.14;` 自体がコンパイルエラー（暗黙の縮小変換を許可しない）。
> C++ は `=` や `()` だと通ってしまうので、`{}` を使って C# と同等の安全性を確保する。
> C# の初期化は `int x = 0;` の1種類だけなので、この使い分けは C++ 特有。

### ラムダ式

```cpp
auto f = [](auto x) { return x * 2; };
std::cout << f(5);  // 10
```

- `[]` : キャプチャリスト（外側の変数を取り込む）
- `(auto x)` : 引数リスト
- `{ ... }` : 関数本体
- `auto` を使うと型を自動推論してくれる
- main 関数内でも定義可能（通常の関数は main 内に定義できない）

> **C#との比較:**
> ```
> // C#
> Func<int, int> f = x => x * 2;
>
> // C++
> auto f = [](int x) { return x * 2; };
> ```
> - C# の `=>` が C++ では `[](){}` 構文になる
> - C++ の `[]` キャプチャリストは C# にない概念。C# のラムダは外側の変数を自動キャプチャするが、
>   C++ では `[&]`（参照）や `[=]`（コピー）で明示的に指定する必要がある
> - C# の `Func<T>` / `Action<T>` に相当するのが `std::function<T>`

### main 関数の戻り値

- `int main()` の `return 0;` は省略可能（暗黙で `return 0;` が挿入される）
- `void main()` は **C++ 規格違反**（一部のコンパイラでは通るが非標準）

## 補足

- `{}` 初期化は C++11 以降で導入。モダン C++ では基本的にこれを使う
- ラムダ式も C++11 以降。`auto` 引数は C++14 以降（ジェネリックラムダ）
- ラムダ式は STL のアルゴリズム（`std::sort`, `std::for_each` 等）と組み合わせて使うことが多い
